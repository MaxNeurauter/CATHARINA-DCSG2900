### 4. Weaknesses & Risks (Cons)  
- **Critical Security Gaps**:  
  - **No firewalls** at *any* layer (Aggregation, Core, or server access). All 16 servers exist in a **single, unsegmented broadcast domain**, enabling lateral movement by attackers.  
  - **No DMZ**: No isolated zone for public-facing servers (e.g., web servers), exposing critical assets to external threats.  
- **Excessive Redundancy (Inefficient)**:  
  - 2 Core routers fully meshed to 8 Aggregation switches (16 uplinks each) creates **over-provisioning** for the 16 servers (only 16 servers require access). This adds unnecessary cost/complexity.  
- **Lack of Security Boundaries**:  
  - No VLANs or access control lists (ACLs) are evident in the diagram → **all servers share the same security context**. A breach in one server compromises all others.

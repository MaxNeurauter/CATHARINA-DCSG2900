#### **Purpose & Architecture**  
The intended design aims to:  
1. Provide external access to internal resources (via `Router0` → `ASA0` → `Switch2`).  
2. Host a DMZ for public-facing services (e.g., `Server-PT` and `PC-PT (PC3)`).

- `ASA0` is positioned only to filter external traffic, **not internal/DMZ traffic**.  

---

#### **Strengths**  
1. **DMZ Presence**: Provides a dedicated segment for public-facing services.  
2. **Firewall (ASA0)**: Acts as a barrier for external access.  
3. **Wireless Access**: `AccessPoint-PT` enables mobile device connectivity.  

---

#### **Critical Weaknesses**  
1. **Single Points of Failure (SPOF)**:  
   - `Switch2` is the central hub for all internal traffic. If it fails, the entire internal network (including PCs, printers, and DMZ) becomes unreachable.  
   - `ASA0` lacks redundancy (no active-standby configuration), so a single failure blocks external access.  

2. **Insecure DMZ Segmentation**:  
   - `Switch1` (DMZ) is physically connected to `Switch2` (internal), creating a **shared broadcast domain**. This allows internal devices (e.g., `PC0-2`) to access DMZ resources **without firewall rules**, increasing attack surface.  
   - No VLANs or layer-3 segmentation, leading to broadcast storms and poor performance.  

3. **Lack of Redundancy**:  
   - No backup paths for critical links (e.g., `Switch2 → ASA0`, `Switch2 → Switch1`).  
   - `Router0` and `Switch0` have no redundant connections, risking external connectivity failures.  

4. **Inefficient Layer 2 Design**:  
   - `Switch2` (an access layer switch) handles all internal traffic without VLANs, causing excessive broadcast traffic and network congestion.  

---

1. **Implement VLANs and Layer 3 Segmentation**

     - VLAN 20: DMZ (`Server-PT`, `PC3`).  
     - VLAN 30: Printers.  
   - **Route between VLANs** using the `ASA0` or a Layer 3 switch. This ensures:  
     - DMZ devices are **isolated from internal networks** (e.g., `PC0-2` cannot directly access `Server-PT`).  
     - Broadcast domains are minimized, improving performance.

2. **Add Redundancy and HA Pairs**

   - **Firewall Redundancy**:  
     - Deploy `ASA0` as **active-standby** (or use a load-balanced setup) to prevent single points of failure.

- **Security**: Prevents unauthorized access to internal systems via the DMZ.

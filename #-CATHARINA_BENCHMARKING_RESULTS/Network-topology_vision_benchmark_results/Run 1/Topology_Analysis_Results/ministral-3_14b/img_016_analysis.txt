### **4. Weaknesses & Risks (Cons)**
❌ **Single Point of Failure (SPOF) Risks:**
   - **Switch0** is the sole point of connectivity for all PCs. If it fails, the entire network segment goes down.
   - **Router3** is the only uplink to external networks. If it fails, all PCs lose internet/WAN access.

❌ **No Redundancy:**
   - No backup links (e.g., secondary uplink, failover routing).
   - No **Spanning Tree Protocol (STP)** configuration is visible (though 2960 switches typically run it by default).

❌ **Lack of Security & Segmentation:**
   - All PCs are in a **single broadcast domain** (no VLANs), increasing collision and attack surfaces.
   - No firewall, ACLs, or traffic filtering is visible.
   - No **DMZ, guest network, or segmentation** for different security zones.

❌ **No High Availability (HA) or Load Balancing:**
   - No **VRRP/HSRP** for router failover.
   - No **stacking or redundant switches** for switch resilience.

❌ **Potential for Broadcast Storms:**
   - Since all devices are on the same subnet, broadcast traffic (e.g., ARP, DHCP) floods the entire network.

---

### **5. Recommendations (Industry Best Practices)**
#### **1. Implement VLAN Segmentation & Access Control Lists (ACLs)**
   - **Action:** Divide PCs into separate VLANs (e.g., VLAN 10 for general use, VLAN 20 for management, VLAN 30 for guests).
   - **Benefits:**
     - Reduces broadcast domains.
     - Enables **micro-segmentation** for security.
     - Allows **role-based access control (RBAC)** via ACLs on the router/switch.
   - **Implementation:**
     - Configure **SVIs (Switch Virtual Interfaces)** on **Switch0** for each VLAN.
     - Apply **inter-VLAN routing** on **Router3** (if using a multilayer switch, this could be done on the switch itself).

#### **2. Introduce Redundancy (Router & Switch Failover)**
   - **Action:**
     - **Add a second uplink** from **Switch0** to a **backup router** (or a second WAN link on **Router3**).
     - Implement **VRRP/HSRP** for router failover.
     - Enable **STP (Spanning Tree Protocol)** to prevent loops if additional switches are added.
   - **Benefits:**
     - Eliminates SPOF for upstream connectivity.
     - Ensures **99.9%+ uptime** in case of hardware failure.
   - **Implementation:**
     - Configure **HSRP** between **Router3** and a backup router.
     - Use **PortFast + BPDU Guard** on **Switch0** to prevent loops.

#### **3. Deploy Basic Security Measures (Firewall & Traffic Filtering)**
   - **Action:**
     - Enable **Cisco IOS Firewall** (if on Router3) or deploy a **basic firewall appliance** (e.g., pfSense, Cisco ASA).
     - Implement **ACLs** to restrict unnecessary traffic (e.g., block ICMP, limit broadcast storms).

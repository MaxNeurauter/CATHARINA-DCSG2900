### 3. Strengths (Pros)  
- **Clear Hierarchical Segmentation:** Separates core (Campus Backbone), distribution (Building Backbone), and access (Workgroups) layers, aligning with Cisco 3-Tier design for scalability.  
- **Redundancy in Core:** Campus Backbone has dual routers (reducing SPOF at the campus level).

### 4. Weaknesses & Risks (Cons)  
- **Single Points of Failure (SPOF):**  
  - *Building Backbone:* Only **1 switch** (no redundant path to Campus Backbone). If it fails, all workgroups go offline.  
  - *Remote Access:* Telecommuter/Remote User connections lack redundant WAN links (1 router per site → no failover).  
- **Security Gaps:**  
  - **No firewalls/ASAs:** Critical boundaries (e.g., Campus → WAN, WAN → Remote Sites) lack security appliances. Remote users directly connect to WAN routers (e.g., "Remote User" router), exposing the network to unfiltered traffic.  
  - **No DMZ:** No isolated zone for public-facing servers (e.g., servers in Enterprise Campus connect directly to Campus Backbone).  
- **Inefficient Design:**  
  - *Unnecessary complexity:* The "WAN" cloud shows 4 routers, but remote sites (Branch, Telecommuter) connect directly to the WAN (1:1), creating asymmetric routing.  
  - *Over-fragmented remote access:* Telecommuter and Remote User use separate routers for similar access needs (no unified access point).  

---

### 5. Recommendations  
1. **Add HA Pairs for Critical SPOF Devices:**  
   - Implement **redundant switches in the Building Backbone** (e.g., dual switches with HSRP/VRRP) and **redundant WAN routers for remote sites** (e.g., dual routers at Branch Office for 1:1 failover). *Reduces SPOF risks.*  
2. **Deploy Security Boundaries:**  
   - Place **firewalls (e.g., Cisco ASA) at 3 key points**:  
     - Between Campus Backbone and WAN (for campus-to-remote traffic).  
     - At the edge of the Enterprise Campus (for server access).  
     - For remote users (e.g., unified remote access via a secure gateway). *Mitigates security exposure.*  
3. **Enhance Segmentation with VLANs:**  
   - Implement **VLANs for workgroups/servers** (e.g., separate VLANs for admin, user, and server traffic) within the Campus and Building Backbones. *Improves security, reduces broadcast domains, and enforces policy control.*

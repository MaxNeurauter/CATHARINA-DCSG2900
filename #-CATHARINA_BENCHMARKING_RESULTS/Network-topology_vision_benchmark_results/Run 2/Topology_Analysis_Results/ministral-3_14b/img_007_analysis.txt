### **4. Weaknesses & Risks (Cons)**
❌ **Single Points of Failure (SPOF):**
   - **Internet Router (2901):** If it fails, the entire network loses Internet access.
   - **ASA Firewalls:** If either ASA fails, traffic between LAN/DMZ/Internet may be disrupted (no **HA pair** configured).
   - **Core Switch (2960-24TT in LAN):** If it fails, all LAN devices (PCs, IP phones, web server) lose connectivity.

❌ **Lack of Redundant Paths:**
   - No **dual WAN links** (only one Internet connection via 2901).
   - No **spanning-tree (STP) or EtherChannel** for switch redundancy.
   - Firewalls are connected in a **non-redundant chain** (failure in one ASA disrupts traffic).

❌ **Potential Security Gaps:**
   - **Flat VLAN Design:** No explicit VLAN segmentation (all LAN devices appear on a single broadcast domain).
   - **DMZ Trust Model:** If a DMZ server is compromised, it could laterally move to the firewall (though firewalls restrict this).
   - **No Explicit ACLs/Zone-Based Policies:** While ASAs provide security, fine-grained **Access Control Lists (ACLs)** or **micro-segmentation** could enhance security.

❌ **Inefficient Routing & Switching:**
   - No **routing protocol (OSPF/EIGRP/BGP)** for dynamic failover (static routing only).
   - No **VLAN trunking** between switches and firewalls (could lead to misconfiguration risks).

### **Purpose & Architecture**  
- **Goal**: A corporate LAN with internet access, isolated DMZ (for servers), and wireless connectivity.  
- **Design Logic**:  
  - Routers provide redundant internet access.  
  - Switches enable layered network segmentation (core → distribution → access).  
  - ASA1 isolates the DMZ from the internal network (security best practice).  

---

### **Strengths (Pros)**  
1. **Security Segmentation**:  
   - DMZ separated via ASA1 ensures servers/printer are isolated from internal traffic (critical for public-facing services).  
2. **Internet Redundancy**:  
   - Dual routers (Router0, Router1) reduce downtime if one fails.  
3. **Logical Grouping**:  
   - Endpoints are neatly grouped (wired, wireless, DMZ), improving manageability.  

---

### **Weaknesses & Risks (Cons)**  
1. **Critical Single Points of Failure (SPOF)**:  
   - **SW0 (Core Switch)**: Failure disconnects *all* internal segments from the internet/routers (total network outage).  
   - **ASA1 (DMZ Firewall)**: Failure isolates the DMZ entirely (servers/printer become inaccessible).  
2. **Lack of Core Redundancy**:  
   - No redundant path to SW0 (e.g., no secondary core switch or stacking), risking full network collapse.  
3. **Firewall Redundancy Gap**:  
   - ASA2 is present but not integrated into a failover configuration (e.g., no Active-Active/Active-Standby setup), leaving the DMZ vulnerable.  
4. **Layer 2 Risks**:  
   - Without VLANs or Spanning Tree Protocol (STP), broadcast storms or loops could occur in internal segments (e.g., SW1/SW2/SW4).  

---

### **Recommendations**  
1. **Implement Core Switch Redundancy**:  
   - **Action**: Add a secondary core switch (e.g., SW0B) and connect both to routers. Configure **VRRP/HSRP** on routers to ensure failover to the core layer.  
   - **Why**: Eliminates SW0 as an SPOF; if SW0 fails, SW0B maintains connectivity.  
2. **Deploy Firewall High Availability (HA)**:  
   - **Action**: Configure ASA1 and ASA2 in an **Active-Active/Active-Standby HA pair** using Cisco ASA failover features.  
   - **Why**: Ensures continuous DMZ access even if one ASA fails (e.g., ASA1 fails → ASA2 takes over instantly).  
3. **Segment Internal Network with VLANs**:  
   - **Action**: Assign VLANs to internal segments (e.g., VLAN 10 = Laptops, VLAN 20 = PCs) on distribution/access switches (SW1/SW2/SW4).  
   - **Why**: Limits broadcast domains, enhances security, and aligns with Cisco 3-Tier model best practices (e.g., reducing attack surface for internal traffic).  

---

### **Summary**  
This design successfully isolates the DMZ and provides internet redundancy but fails to address **core switch SPOF** and **DMZ firewall redundancy**. By implementing core redundancy, firewall HA, and VLAN segmentation, the network becomes more resilient, scalable, and secure—aligning with enterprise-grade Cisco 3-Tier architecture principles.

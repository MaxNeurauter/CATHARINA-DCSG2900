#### **5. Weaknesses & Risks**  
- **Single Point of Failure (SPOF)**:  
  - **Router1 is the only internet gateway**. If it fails, **all devices lose internet access**.  
  - **Linear switch chain (Switch0 ↔ Switch1 ↔ Switch2)** creates additional SPOF:  
    - Failure of **Switch1** splits the network into two segments (Switch0 ↔ Switch2), isolating devices on Switch2 from Switch0.  
- **Security Vulnerabilities**:  
  - **No segmentation**: All devices (PCs, server, laptop) are in the same broadcast domain, exposing the server to attacks from user PCs (e.g., malware propagation, port scanning).  
- **Broadcast Storm Risk**:  
  - The linear switch interconnection (without Spanning Tree Protocol, STP) risks **broadcast storms** and network congestion, especially with high broadcast traffic.  
- **Scalability Issues**:  
  - Not suitable for larger networks. Adding more devices or segments would require significant reconfiguration (e.g., VLAN implementation or redundant routing).  
- **Inefficient Traffic Management**:  
  - Without VLANs or firewalls, traffic between segments (e.g., PCs ↔ Server) is uncontrolled, leading to potential security breaches and bandwidth waste.  

---

#### **6. Recommendations**  
1. **Implement VLANs**:  
   - Segment the network into distinct broadcast domains (e.g., VLAN for user PCs, VLAN for servers).  
   - **Example**: Assign Switch0 and Switch1 to a "User VLAN" (e.g., VLAN 10), and Switch2 to a "Server VLAN" (e.g., VLAN 20).  
   - *Benefit*: Isolates the server from user traffic, reducing attack surface.  

2. **Deploy Firewalls**:  
   - Place a firewall (e.g., Cisco ASA or software-based) between the User VLAN and Server VLAN.  
   - *Benefit*: Controls traffic flow, blocks unauthorized access to the server.  

3. **Add Router Redundancy**:  
   - Implement **HSRP (Hot Standby Router Protocol)** or **VRRP** for Router1 to ensure failover to a secondary router.  
   - *Benefit*: Eliminates SPOF for internet access.  

4. **Optimize Switch Interconnections**:  
   - Replace the linear chain with a **star topology** or use **STP** to prevent broadcast storms.  
   - *Benefit*: Avoids network fragmentation and broadcast storms.  

5. **Enforce Access Control**:  
   - Apply **ACLs (Access Control Lists)** on Router1 to restrict traffic between VLANs (e.g., block ICMP to the server from user PCs).  
   - *Benefit*: Adds granular security policies.  

6. **Upgrade to Hierarchical Design**:  
   - Introduce a **Layer 3 distribution layer** (e.g., a Cisco Catalyst 3560) to route traffic between VLANs.  
   - *Benefit*: Enables scalability and better traffic management for larger networks.

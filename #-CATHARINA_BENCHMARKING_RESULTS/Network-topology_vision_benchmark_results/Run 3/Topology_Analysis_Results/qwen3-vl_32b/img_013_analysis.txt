### **4. Weaknesses & Risks (Cons)**  
- **Critical Security Gaps**:  
  - **No firewalls/ASAs** between layers → No security boundaries (e.g., lack of DMZ, east-west traffic control).  
  - **No VLAN segmentation** shown → All endpoints (PCs/printers) likely share a single broadcast domain per L2 switch.  
- **Single Points of Failure (SPOFs)**:  
  - **Distribution Layer**: If one L3 switch fails, 2 access segments (4 L2 switches) lose connectivity (though partially mitigated by dual uplinks).  
  - **Access Layer**: L2 switches lack redundancy *within* the access layer (e.g., no stacking/stacking).  
- **Inefficient Design**:

  - Over-reliance on L3 switches for access layer: Access layer should be L2, but L2 switches are labeled "Remote" (implying potential WAN connectivity without defined security).

### **5. Recommendations**

### 4. **Weaknesses & Risks (Cons)**  
- **Critical Single Points of Failure (SPOFs)**:  
  - **Router1**: Failure of 2901 Router1 **cascades to the entire network** (no HA pair or backup route).  
  - **Switch Chain**: If *any* switch in the daisy chain (e.g., Switch0 or Switch1) fails, all downstream devices lose connectivity.  
- **No Security Boundaries**:  
  - **Server0 is exposed**: The server shares the same Layer 2 segment as all clients, enabling direct access from any endpoint (e.g., PC0, Laptop0) without firewalls or ACLs. This violates the principle of least privilege.  
  - **No VLANs**: A single broadcast domain increases risk of ARP poisoning, broadcast storms, and lateral movement of threats.  
- **Inefficient Topology**:  
  - The linear switch chain (Switch0 → Switch1 → Switch2) creates a **bottleneck** and **unreliable path** (e.g., traffic between PC0 and Laptop0 must traverse all 3 switches).  
  - **No Redundancy**: All links are single-homed, leading to network outages if any physical link fails.  

---

### 5. **Recommendations**  
1. **Implement VLAN Segmentation (Cisco 3-Tier Model)**:  
   - Create a **VLAN for servers** (e.g., Server0 on VLAN 10) and **clients on separate VLANs** (e.g., PCs on VLAN 20, laptops on VLAN 30).  
   - Use **Layer 3 routing** (e.g., enable IP routing on the 2901 router or add a core switch) to control inter-VLAN traffic. *This isolates the server, reduces broadcast domains, and improves security*.  

2. **Add Redundancy to Critical Components**:  
   - **Switch Redundancy**: Replace the linear daisy chain with a **star topology** where all switches connect directly to a **core switch** (e.g., 3560/3750) or the router. Use **HSRP/VRRP** for gateway redundancy.  
   - **Router HA Pair**: Deploy a second 2901 router with **HSRP** (or a Cisco ASA firewall) to eliminate the SPOF at the internet edge. *This ensures continuous connectivity if Router1 fails*.  

3. **Enforce Security Boundaries**:  
   - Place **Server0 in a DMZ** (e.g., by adding a firewall between the router and Switch2) and apply **inbound ACLs** to restrict access to the server.  
   - Use **802.1X** or **port security** on switches to control endpoint access. *This mitigates exposure of the server to untrusted devices*.

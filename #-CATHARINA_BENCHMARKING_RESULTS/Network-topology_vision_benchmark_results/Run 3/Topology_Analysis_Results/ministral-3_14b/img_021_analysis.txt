### **5. Recommendations (Industry Best Practices)**
#### **🔹 Recommendation 1: Implement Redundancy & High Availability**
   - **Add redundant firewalls (ASA HA pair)** to eliminate SPOF.
   - **Deploy HSRP/VRRP** between switches for failover.
   - **Add redundant uplinks** (e.g., Switch0 → Router & ASA with **STP/MSTP** for loop prevention).
   - **Example:**
     ```
     Switch0 (Primary) --| HSRP |-- Switch1 (Secondary)
                          |
                     ASA0 (Primary) --| Failover |-- ASA1 (Secondary)
     ```

#### **🔹 Recommendation 2: Apply VLAN Segmentation & Access Control**
   - **Create VLANs** for:
     - **VLAN 10** – Internal PCs (PC0, PC1, PC2)
     - **VLAN 20** – VoIP (if applicable)
     - **VLAN 30** – IoT/Printers
     - **VLAN 40** – DMZ (Only Server0, **remove PC3**)
   - **Configure ACLs on ASA/Switches** to restrict inter-VLAN traffic.
   - **Use 802.1X authentication** for wired/wireless devices.

#### **🔹 Recommendation 3: Adopt a Cisco 3-Tier Hierarchical Model**
   - **Core Layer (Router0):** High-speed routing (ISP connection).
   - **Distribution Layer (Switch0 + Redundant Switch):** Policy enforcement (ACLs, QoS, VLAN routing).
   - **Access Layer (Switch1, Switch2):** Endpoint connectivity (PCs, AP, Printer).
   - **Benefits:**
     - **Scalability** (easy to add new VLANs/switches).
     - **Fault isolation** (failures in Access layer don’t crash Core).
     - **Simplified troubleshooting** (layered design).

#### **🔹 Bonus: Security Hardening**
   - **Disable unused ports** on switches.
   - **Enable port security** to prevent MAC flooding.
   - **Segment guest wireless (Access Point)** into a separate VLAN.
   - **Monitor DMZ traffic** with IDS/IPS (e.g., Cisco Firepower).

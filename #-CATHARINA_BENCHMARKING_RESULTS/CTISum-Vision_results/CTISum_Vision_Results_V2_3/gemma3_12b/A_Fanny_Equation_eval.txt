**CTIS Summary: Fanny Malware (USB Worm)**

**1. Threat Actor & Attribution:**

*   **Likely Attribution:** The Fanny malware is attributed to the same group, or a closely related entity, as the creators of Stuxnet. Evidence includes shared coding techniques, similar architecture, and the use of unique magic numbers and code patterns.
*   **Motivation:** Likely espionage and mapping of air-gapped networks.

**2. Threat Landscape & Target:**

*   **Targeted Environments:** Primarily targets air-gapped systems, isolated networks. The initial infection relies on a user plugging in a USB drive.
*   **Geographic Focus:** Pakistan is disproportionately affected, comprising the vast majority of known infections. Other affected countries include Indonesia, Vietnam, China, Bangladesh, Nigeria, and Malaysia.
*   **Impact:** Potential for reconnaissance, data exfiltration, and mapping of networks for future attacks.

**3. TTPs (Tactics, Techniques, and Procedures):**

*   **Initial Access:** Exploits user interaction – infected USB drive inserted into a computer.
*   **Persistence:** Creates hidden container using a custom FAT/FAT32 filesystem driver within the USB drive’s seemingly normal structure.
*   **Discovery:**  Detects new USB drives and allocates space for hidden storage.
*   **Credential Access:** Collects system information (OS version, user name, process list, etc.).
*   **Command & Control (C2):** Communicates with a sinkhole server to exfiltrate collected data and potentially receive commands.
*   **Evasion:** Masks itself to resemble legitimate software, mimicking actions like adding files to AutoRun to avoid detection as malware.

**4. Technical Details:**

*   **Malware Type:** Custom-built USB worm with reconnaissance capabilities.
*   **File Types:**
    *   `fanny.bmp` (Deception – seemingly benign image file used as a carrier)
    *   `agentcpd.dll` (USB Backdoor DLL – responsible for creating the hidden container and performing reconnaissance).
*   **Custom FAT Driver:** Uses a non-standard FAT driver to create a hidden file system on the USB drive, making the container undetectable by standard tools.
*   **Hidden Storage:**  Data is stored in a hidden container on the USB drive, making it appear to be an empty drive to the user.
*   **File Manipulation:**  Icon of the target drive are hidden.

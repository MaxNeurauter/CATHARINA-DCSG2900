**CTIS: Fanny Malware - Targeted Reconnaissance & Data Exfiltration Tool**

**1. Executive Summary:**

Fanny is a sophisticated USB worm and backdoor discovered in 2012, later linked to the Equation Group (the same group behind Stuxnet). It appears to have been used for reconnaissance purposes, specifically mapping air-gapped networks, potentially in preparation for future attacks like Stuxnet. It leverages a combination of USB worm capabilities, rootkit techniques, and a hidden storage mechanism to operate discreetly and exfiltrate data.  The sheer volume of infections and the geographic distribution suggest a wide-ranging, targeted operation.

**2. Malware Overview:**

*   **Name:** Fanny
*   **Family:**  USB Worm / Backdoor / Rootkit
*   **Attribution:** Likely Equation Group (shared infrastructure and tactics with Stuxnet and Flame)
*   **Platform:** Windows
*   **Delivery Mechanism:**  USB drives.  It is a self-propagating worm - once infected, it can spread to other USB drives plugged into the system.
*   **Primary Function:** Reconnaissance and Data Exfiltration. Not believed to be a direct attack tool itself, but a "mapping" tool for future exploitation.

**3. Technical Details:**

*   **Persistence:** Employs rootkit techniques to conceal its presence and maintain persistence on the infected system.  It hides files and processes.
*   **Hidden Storage:**  Creates a hidden, encrypted container on the USB drive.  This hidden storage is used to store:
    *   Configuration data
    *   System information from the infected machine (OS version, user details, hardware details)
    *   Commands to be executed on the air-gapped network.
*   **FAT Modification:** Alters the FAT file system structure on the USB drive to create a hidden partition and directory structure.  This modification is often invisible to standard file system utilities.
*   **Masquerading:** Employs techniques to mimic legitimate system processes and files to avoid detection. This includes deleting system files to appear less suspicious.
*   **Network Communication:** Communicates with a command-and-control (C&C) server to receive instructions and exfiltrate data (though it seems this communication was severely limited/avoided in its operation).

### **3. Attack Path & Technical Impact**
#### **Primary Attack Vector: Kerberos Delegation Abuse**
- **Golden Ticket Attack**:
  If an attacker extracts the user’s **NTLM hash or AES keys** (via tools like Mimikatz or BloodHound), they can forge a **Golden Ticket** for this account, granting **full domain impersonation** (e.g., `krbtgt` or any service account).
  - **Impact**: Full domain compromise, lateral movement, and privilege escalation.

- **Silver Ticket Attack**:
  If the account has access to a **constrained delegation target** (e.g., a SQL server or file share), an attacker could forge a **Silver Ticket** to authenticate as this user to the constrained service.
  - **Impact**: Unauthorized access to sensitive resources (e.g., databases, shared folders).

- **Pass-the-Ticket (PtT) or PtC**:
  The attacker could use the delegated credentials to **pass a Kerberos ticket** to other systems, maintaining persistence.

#### **Secondary Risks**:
- **Lateral Movement**: If the account is in a privileged group (e.g., Domain Admins), the attacker could escalate privileges across the domain.
- **Resource Hijacking**: Impersonation of critical services (e.g., RDP, LDAP, or SMB) could lead to data exfiltration or system takeover.

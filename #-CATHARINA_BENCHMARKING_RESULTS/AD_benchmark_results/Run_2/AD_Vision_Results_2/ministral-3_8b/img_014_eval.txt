### **3. Attack Path (If Vulnerable)**
While the current configuration is not **Critically Vulnerable**, the following attack vectors could be exploited if combined with other misconfigurations:

#### **A. Privilege Escalation via GPO Manipulation**
- If an attacker gains access to a **Domain User** account with **Read** access to the GPO, they could:
  - Modify GPO settings to deploy malicious scripts (e.g., via **Startup Scripts** or **Logon Scripts**).
  - Disable security features (e.g., **Complex Password Policy**, **Account Lockout Threshold**).
  - Deploy **Golden Ticket** or **Silver Ticket** attacks if Kerberos delegation is misconfigured.

#### **B. Kerberoasting (If Combined with Weak Password Policies)**
- If the domain has weak password policies (e.g., short passwords, no complexity), an attacker could perform **Kerberoasting** (abusing **Authenticated Users** access to GPOs to extract service account hashes).

#### **C. Unconstrained Delegation (If Present Elsewhere)**
- If **constrained delegation** is not enforced on domain controllers or critical services, an attacker could use **Golden Ticket** or **Silver Ticket** attacks after compromising a user account.

---

    - **SYSTEM** (Edit settings, delete, modify security)

#### **Step 2: Apply Least Privilege to Domain/Enterprise Admins**
- Ensure **Domain Admins** and **Enterprise Admins** have **only the minimum required permissions**:
  - **Domain Admins**: **Read** + **Apply Group Policy** (No **Modify Security**).

- **Disable inheritance where unnecessary** (if using **Starter GPOs**).
- **Use Security Filtering** to restrict GPO application to specific OUs.

#### **Step 4: Audit and Monitor GPO Changes**
- Enable **Windows Event Log Auditing** for GPO changes (Event ID **1310** for GPO modifications).
- Use **Group Policy Results** to verify compliance.

#### **Step 5: Harden Domain-Wide Security (If Applicable)**

- Enforce **strong password policies** (long passwords, complexity, account lockout).

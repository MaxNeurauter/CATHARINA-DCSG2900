### **3. Attack Paths & Technical Impact**

#### **A. Privilege Escalation via Overly Permissive Rights**
**Policies at Risk:**
1. **Bypass traverse checking**
   - Assigned to: `BUILTIN\Pre-Windows 2000 Compatible Access`, `NT AUTHORITY\Authenticated Users`, `Everyone`
   - **Impact:** An attacker with a standard user account can traverse directories and access files they shouldn’t see (e.g., via `..` path traversal). This can lead to **lateral movement** and **privilege escalation** (e.g., accessing sensitive files in `C:\Windows\System32`).
   - **Attack Vector:** **Local Privilege Escalation** (e.g., exploiting misconfigured ACLs to move laterally).

2. **Backup Operators group**
   - Assigned to: `BUILTIN\Backup Operators` (granted rights like **Back up files and directories**, **Restore files and directories**)
   - **Impact:** Backup Operators can **dump LSASS memory** (via `ntdsutil.exe` or `procdump.exe`) to extract **hashed credentials** (e.g., for **Pass-the-Hash attacks**).
   - **Attack Vector:** **Credential Dumping** (e.g., `mimikatz`, `SecretsDump`).

3. **Log on locally**
   - Assigned to: `NT AUTHORITY\ENTERPRISE DOMAIN CONTROLLERS`, `BUILTIN\Print Operators`, `BUILTIN\Server Operators`, `BUILTIN\Account Operators`, `BUILTIN\Backup Operators`, `BUILTIN\Administrators`
   - **Impact:** `Account Operators` and `Backup Operators` should **not** have this right by default. If compromised, they can **create privileged accounts** or **impersonate other users**.

4. **Force shutdown from a remote system**
   - Assigned to: `BUILTIN\Server Operators`, `BUILTIN\Administrators`
   - **Impact:** If `Server Operators` can force shutdowns, they could **disrupt services** or **trigger crashes** to escalate privileges.

5. **Debug programs**
   - Assigned to: `BUILTIN\Administrators` (correct, but **not assigned to any other groups**)
   - **Impact:** If accidentally assigned to broader groups (e.g., `Authenticated Users`), attackers could **debug LSASS** to extract credentials.

6. **Access this computer from the network**
   - Assigned to: `BUILTIN\Pre-Windows 2000 Compatible Access`, `NT AUTHORITY\ENTERPRISE DOMAIN CONTROLLERS`, `NT AUTHORITY\Authenticated Users`, `BUILTIN\Administrators`, `Everyone`
   - **Impact:** `Everyone` having this right means **any user** can **RDP or network logon**, increasing the attack surface for **brute-force or credential stuffing**.

7. **Load and unload device drivers**
   - Assigned to: `BUILTIN\Print Operators`, `BUILTIN\Administrators`
   - **Impact:** `Print Operators` should **not** have this right. If compromised, they could **load malicious drivers** (e.g., **kernel-mode rootkits**).

#### **Additional Hardening Steps:**
1. **Create a Custom Admin Group**:
   - Replace `BUILTIN\Administrators` with a **restricted admin group** (e.g., `Domain Admins` with **just-in-time (JIT) privileges**).
   - Remove `BUILTIN\Administrators` from all rights and reassign to the custom group.

2. **Enable Least Privilege for Service Accounts**:
   - Ensure `NT AUTHORITY\Network Service` and `NT AUTHORITY\Local Service` are **not** assigned unnecessary rights.

3. **Audit and Monitor**:
   - Enable **Windows Event Logs** for **Security ID (SID) changes** and **user right assignments**.
   - Use **Group Policy Preferences (GPP)** to **deny** rights at the user level where possible.

4. **Disable Unused Built-in Accounts**:
   - Disable `Guest`, `Administrator`, and other default accounts.

5. **Implement Just-In-Time (JIT) Privileges**:
   - Use tools like **CrowdStrike Falcon**, **Microsoft Defender for Identity**, or **Privailege Management for Active Directory (PAM)** to grant elevated rights **only when needed**.

6. **Regularly Audit User Rights**:
   - Run **`gpresult /h report.html`** and **`secpol.msc`** to verify assignments.
   - Use **PowerShell** to check assignments:
     ```powershell
     Get-LocalUser -LocalOnly | Get-LocalGroupMember | Where-Object {$_.Group -like "*Backup Operators*"}
     ```

7. **Block Unnecessary Protocols**:
   - Disable **SMBv1**, **RDP if not needed**, and **unencrypted protocols**.

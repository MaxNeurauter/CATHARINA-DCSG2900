### **3. Attack Paths and Technical Impact**
#### **A. Kerberoasting (High Risk)**
- **Vector**:
  - Attackers enumerate service accounts (e.g., `jon.snow`, `brandon.stark`) in the `Domain Users` or `Enterprise Admins` groups.
  - Extract hashed Kerberos tickets (via AS-REP or Service Principal Names) and crack offline using tools like **RocketKit** or **Hashcat**.
- **Impact**:
  - Compromised service accounts can be used for lateral movement or privilege escalation.
  - If `krbtgt` is compromised, attackers can forge Golden Tickets.

#### **B. AS-REP Roasting (High Risk)**
- **Vector**:
  - Attackers target accounts with **"Do not require Kerberos pre-authentication"** enabled (not visible here but often misconfigured).
  - Extract unencrypted AS-REP tickets (even if passwords are complex) and crack offline.
- **Impact**:
  - Full account compromise if passwords are weak or reused.

#### **C. Privilege Escalation (Critical Risk)**
- **Vector**:
  - Fictional names (`jon.snow`, `arya.stark`) in `Domain Admins` or `Enterprise Admins` groups.
  - Attackers exploit weak passwords or session hijacking to gain domain admin rights.
- **Impact**:
  - Full domain compromise (data exfiltration, ransomware, or lateral movement).

#### **D. Pass-the-Hash (High Risk)**
- **Vector**:
  - Weak or default passwords (e.g., `Guest` account) are cracked via brute force or phishing.
  - Hashes are passed to other systems for unauthorized access.
- **Impact**:
  - Unauthorized access to all systems the account has permissions for.

#### **E. RODC Compromise (Medium Risk)**
- **Vector**:
  - Misconfigured `Allowed RODC Password Replication` group contains non-admin accounts (e.g., `arya.stark`).
  - Attackers replicate passwords to RODCs, bypassing WORM (Write Once, Read Many) protection.
- **Impact**:
  - Compromised RODCs can be used to reset passwords for all domain accounts.

### **4. Remediation Steps**
#### **Step 1: Disable or Clean Up Test/Placeholder Accounts**
- **Action**:
  - Disable or delete fictional accounts (`jon.snow`, `arya.stark`, `brandon.stark`, `hodor`, etc.).
  - Audit all user accounts for unusual names or test accounts.
- **Tool**:
  ```powershell
  Get-ADUser -Filter * -Properties Name | Where-Object { $_.Name -like "*snow*" -or $_.Name -like "*stark*" } | Disable-ADAccount
  ```

#### **Step 2: Enforce Strong Password Policies**
- **Action**:
  - Set minimum password length (12+ characters).
  - Enforce password complexity (uppercase, lowercase, numbers, symbols).
  - Enable password expiration (90 days max).
  - Disable "Password never expires" for all accounts.
- **Tool**:
  ```powershell
  Set-ADDefaultDomainPasswordPolicy -EnforcePasswordHistory 24 -MaxPasswordAge 90 -MinPasswordLength 12 -MinPasswordAge 1
  ```

#### **Step 3: Enable Multi-Factor Authentication (MFA)**
- **Action**:
  - Enforce MFA for all privileged accounts (`Domain Admins`, `Enterprise Admins`, `krbtgt`).
  - Use Microsoft Authenticator, Duo, or similar.
- **Tool**:
  - Configure via **Microsoft Entra ID (Azure AD)** or **Conditional Access Policies**.

#### **Step 4: Apply Least Privilege Principle**
- **Action**:
  - Remove non-essential accounts from `Domain Admins`, `Enterprise Admins`, and `Allowed RODC Password Replication`.
  - Use **Group Policy Preferences (GPP)** or **Restricted Groups** to enforce membership.
- **Tool**:
  ```powershell
  Remove-ADGroupMember -Identity "Domain Admins" -Members "jon.snow", "arya.stark"
  ```

#### **Step 5: Disable the Guest Account**
- **Action**:
  - Disable the built-in `Guest` account.
  - Remove any lingering guest permissions.
- **Tool**:
  ```powershell
  Disable-ADAccount -Identity "Guest"
  ```

#### **Step 6: Enforce Account Lockout Policies**
- **Action**:
  - Set lockout threshold (e.g., 5 failed attempts).
  - Set lockout duration (e.g., 30 minutes).
  - Reset lockout after successful login.
- **Tool**:
  ```powershell
  Set-ADDefaultDomainPasswordPolicy -LockoutThreshold 5 -LockoutDuration 30 -ResetLockoutCountAfter
  ```

#### **Step 7: Audit and Monitor for Suspicious Activity**
- **Action**:
  - Enable **Advanced Audit Policies** (e.g., "Audit Logon Events," "Audit Privilege Use").
  - Use **Microsoft Sentinel** or **SIEM tools** to detect anomalies.
- **Tool**:
  ```powershell
  Auditpol /set /subcategory:"Logon" /success:enable /failure:enable
  ```

#### **Step 8: Review and Remove Unnecessary Service Accounts**
- **Action**:
  - Audit all service accounts (`krbtgt`, `SPNs`) for unused or overprivileged accounts.
  - Restrict permissions to least privilege.
- **Tool**:
  ```powershell
  Get-ADUser -Filter { ServicePrincipalName -ne "$null" } | Select-Object Name, ServicePrincipalName
  ```

#### **Step 9: Implement RODC Best Practices**
- **Action**:
  - Ensure only necessary accounts are in `Allowed RODC Password Replication`.
  - Verify `Denied RODC Password Replication` contains only necessary accounts.
- **Tool**:
  ```powershell
  Get-ADGroupMember -Identity "Allowed RODC Password Replication" | Select-Object Name
  ```

#### **Step 10: Educate Users on Security Hygiene**
- **Action**:
  - Train users on password hygiene, phishing awareness, and MFA usage.
  - Enforce password rotation policies.

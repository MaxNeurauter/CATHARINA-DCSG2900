### **3. Attack Paths & Technical Impact**
#### **A. Privilege Escalation via Password Policy Manipulation**
- **Attack Vector:** An attacker with **Authenticated Users** access could:
  1. **Disable password expiration** (`Unexpire password`).
  2. **Disable password complexity requirements** (`Update password not required bit`).
  3. **Set a weak password** (if they can reset it via other means).
- **Impact:**
  - Persistent access without detection (no password rotation).
  - Ability to **Kerberoast** or **AS-REP Roast** if combined with other misconfigurations.

#### **B. Unintended Access via Legacy Permissions**
- **Attack Vector:** If `Pre-Windows 2000 Compatible Access` is misconfigured, an attacker could:
  1. Gain **unexpected read/write access** to descendant objects.
  2. Exploit **inheritance conflicts** to escalate privileges.

#### **C. Potential for Object Tampering**
- **Attack Vector:** If an attacker gains **Authenticated Users** access, they could:
  1. **Modify SPNs** (Service Principal Names) for **Kerberoasting**.
  2. **Modify account attributes** to bypass access controls.

---

### **4. Remediation Steps**
#### **Step 1: Restrict Password Policy Modifications**
- **Remove `Authenticated Users` permissions for:**
  - `Unexpire password`
  - `Update password not required bit`
  - `Enable per-user reversibly encrypted password`
- **Grant only to `Domain Admins` or `Enterprise Admins`.**

#### **Step 2: Remove Legacy Permissions**
- **Remove `Pre-Windows 2000 Compatible Access`** for descendant objects unless absolutely necessary.

#### **Step 3: Review Inheritance & Apply Explicit Deny**
- **Ensure no unnecessary inheritance** is granting excessive permissions.
- **Use `View Advanced`** to check for conflicting permissions.

#### **Step 4: Apply Least Privilege**
- **Remove `Authenticated Users` from sensitive actions** (e.g., password policy changes).
- **Ensure only `Domain Admins`/`Enterprise Admins`** can modify critical attributes.

#### **Step 5: Audit & Monitor**
- **Check Event Logs** for unusual permission changes (`Security Event ID 4663` for permission modifications).
- **Use `Accessdenied` or `Audit Object Access` GPO** to log permission changes.

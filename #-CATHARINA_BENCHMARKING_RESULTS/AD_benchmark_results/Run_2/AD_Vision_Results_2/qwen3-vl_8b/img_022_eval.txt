### **Attack Path**  
**Kerberoasting** (a password-based Kerberos attack) is the primary threat:  
- **How it works**: An attacker requests service tickets for user accounts and cracks them using password dictionaries or brute-force tools.  
- **Technical impact**:  
  - Weak passwords (5 chars, no complexity) allow attackers to rapidly crack service tickets.  
  - Passwords never expiring mean compromised accounts remain accessible for years, enabling **persistent access to network resources** (e.g., file servers, domain controllers).  
  - Attackers gain **unrestricted access** to critical systems (e.g., domain controllers, file shares) by leveraging cracked Kerberos tickets.  

---

### **Remediation Steps**  
Follow these steps to secure password policies:  

1. **Open Group Policy Management** (GPM):  
   - Navigate to `Group Policy Management Console` → `Forest` → `Domain` → `Group Policy Objects`.  

2. **Edit the relevant GPO** (e.g., `Default Domain Policy`):  
   - Right-click the GPO → `Edit`.  
   - Go to `Computer Configuration` → `Policies` → `Windows Settings` → `Security Settings` → `Account Policies` → `Password Policy`.  

3. **Fix critical settings**:  
   - **Maximum password age**:  
     - Set to **90 days** (or a shorter, secure value like 60 days).  
     - *Why?* Prevents passwords from being valid indefinitely.  
   - **Minimum password length**:  
     - Set to **12 characters**.  
     - *Why?* 12+ characters significantly increase brute-force difficulty.  
   - **Password complexity**:  
     - Set to **Enabled**.  
     - *Why?* Enforces uppercase, lowercase, numbers, and symbols.  
   - **Enforce password history**:  
     - Verify it is set to **24** (standard).  

4. **Verify and enforce changes**:  
   - Click `OK` → `Apply` → `Close` the GPO editor.  
   - Force a Group Policy refresh:  
     - Run `gpupdate /force` on domain-joined machines.  

5. **Validate**:  
   - Check password requirements via `Control Panel` → `User Accounts` → `Change your password`.  
   - Confirm passwords meet complexity rules and expire within 90 days.  

---

### **Why This Works**  
- **Kerberoasting is mitigated** by enforcing strong passwords and short expiration cycles.  
- **Persistent access is prevented** by ensuring passwords are changed regularly (no 102-year validity).  
- **Attack surface is reduced** by eliminating weak password patterns.  

> **Note**: If the GPO applies to specific user groups, ensure changes target the correct scope (e.g., `Domain Controllers` or `Users`), not the entire domain.  

By addressing these vulnerabilities, the system becomes resilient against Kerberoasting, password brute-force attacks, and persistent access scenarios.

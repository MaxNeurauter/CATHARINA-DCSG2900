  - `Enterprise Admins` and `Administrators` have broad "Full control" / "Special" access (applies to object + descendants—standard for enterprise-level permissions).  

### 2. Assessment  
**Critically Vulnerable**. The `Everyone` group having **"Change password"** access on a computer object is a high-severity misconfiguration.  

### 3. Attack Path  
- **Attack Vector**: **Privilege Escalation** (via **Computer Account Password Theft/Reset**).  
- **Technical Impact**:  
  - The `Change password` permission on a computer object allows *any user in the domain* (or even anonymous users, if exposed) to reset the computer account’s password.  
  - An attacker (e.g., a low-privilege user) can:  
    1. Reset the computer account’s password to a known value.  
    2. Impersonate the computer account (e.g., via `mimikatz` or `RDP`/`SMB` authentication).  
    3. Exploit the computer account to access network resources (e.g., file shares, printers) or escalate to **Local Admin** on the machine (if the computer account has local admin rights, which is common for domain-joined machines).  
  - This directly enables **lateral movement** and **persistence** (e.g., an attacker could set up a backdoor by exploiting the computer account’s network identity).  

### 4. Remediation  
Follow these steps to fix the misconfiguration:  
1. **Open Advanced Security Settings**:  
   - In Active Directory Users and Computers (ADUC), locate the computer object `WINTERFELL`.  
   - Right-click → **Properties** → **Security** tab → **Advanced**.  
2. **Locate the "Everyone" Entry**:  
   - In the **Permission entries** list, find the row for `Everyone` (with "Change password" access).  
3. **Remove the Permission**:  
   - Select the `Everyone` entry → Click **Remove** (or **Edit** → uncheck "Change password" → **OK**).  
4. **Verify the Fix**:  
   - Ensure *only* `Domain Admins` (or a highly restricted admin group) retains "Change password" access (this is the default in a secure AD environment).  
5. **Prevent Recurrence**:  
   - Review all computer objects for similar misconfigurations (use PowerShell: `Get-ADComputer -Filter * | Get-Acl | Where-Object { $_.Access | Where-Object { $_.AccessRights -eq "ChangePassword" } -and $_.AccessIdentity -eq "Everyone" }`).  
   - Enforce group policy to restrict "Change password" access to `Domain Admins` only (via Group Policy Objects).  

> **Note**: This issue is a known critical flaw in AD (e.g., covered in [Microsoft’s "Hardening Windows" guidance](https://learn.microsoft.com/en-us/windows-server/identity/ad-ds/plan/security-best-practices) and [SANS SEC504](https://www.sans.org/courses/active-directory-forensics/)). The "Everyone" group **must never** have "Change password" access on computer objects.

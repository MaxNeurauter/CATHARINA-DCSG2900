If "Everyone" has "Add Members" rights, *any user in the domain* (or an attacker who compromises a low-privilege account) can add themselves to the Domain Admins group.  

### 3. Attack Path  
- **Vector**: **Privilege Escalation** via **Group Membership Manipulation**.  
- **Technical Impact**:  
  - An attacker compromises a standard user account (e.g., via phishing, vulnerability exploitation).  
  - Using tools like `Add-ADGroupMember` (PowerShell) or `dsmod` (command line), the attacker **adds their own account to the Domain Admins group**.  
  - This grants the attacker **full control over the domain** (e.g., creating new admin accounts, disabling security policies, exfiltrating data, or deploying ransomware).  
  - This is *not* Kerberoasting or Silver Ticket (which target user accounts), but a direct **"gates" vulnerability** where the attacker bypasses all security controls by *abusing group membership permissions*.

### 2. Assessment  
**Critically Vulnerable**  

### 3. Attack Path  
- **Attack Vector**: **Unconstrained Delegation** (a well-known AD vulnerability).  
- **Technical Impact**:  
  - Unconstrained delegation allows a compromised user account to request **TGS (Ticket-Granting Service) tickets for *any* service** in the domain using the user’s TGT (Ticket-Granting Ticket).  
  - An attacker can:  
    1. Steal the user’s TGT (e.g., via pass-the-ticket, pass-the-hash, or by compromising the user’s session).  
    2. Use the TGT to request service tickets for **any critical service** (e.g., Domain Controllers, file servers, databases).  
    3. Impersonate the user to access resources, escalate privileges, or move laterally across the domain.  
  - This effectively grants the attacker **"bypass all access controls"** for the user’s permissions, as the delegation is not restricted to specific services.  

### 4. Remediation  
**Step-by-Step Fix**:  
1. **Open Active Directory Users and Computers (ADUC)**.  
2. **Locate the user account** (`sansa.stark`) and right-click → **Properties**.  
3. **Navigate to the "Delegation" tab**.  
4. **Select "Do not trust this user for delegation"** (the default, most secure setting for most users).  
   - *If the user requires limited delegation (e.g., for specific services), select "Trust this user for delegation to specified services only" and add only the necessary services via "Add…".*  
5. **Click "OK"** to apply changes.  

**Verification**:  
- Recheck the Delegation tab to ensure the correct option is selected.  
- Monitor the domain for any unexpected delegation privileges (e.g., via `Get-ADUser -Filter * -Properties TrustedForDelegation`).  

> **Note**: Unconstrained delegation is **only appropriate for highly privileged service accounts** (e.g., domain controllers) and **never for user accounts**. For standard users, **"Do not trust this user for delegation"** is the only secure configuration.

### **Attack Paths & Technical Impact**
#### **1. Kerberoasting (Critical)**
- **Attack Vector**: If `krbtgt` has a weak password, attackers can request Service Principal Names (SPNs) for any user (e.g., `sql_svc`) and perform offline AS-REP roasting.
- **Impact**:
  - Brute-force the hash to obtain plaintext credentials.
  - Lateral movement to domain controllers if `sql_svc` has high privileges.
  - Persistence via Golden Ticket attacks if `krbtgt` is compromised.

#### **2. Privilege Escalation (High)**
- **Attack Vector**: If `Domain Admins` or `Group Policy Creator Owners` are misused (e.g., via Pass-the-Hash or stolen credentials), attackers gain full domain control.
- **Impact**:
  - Full domain takeover.
  - Installation of backdoors (e.g., via Group Policy).

#### **3. AS-REP Roasting (Medium-High)**
- **Attack Vector**: If users (e.g., `jeor.mormont`, `arya.stark`) are not marked as **"Account is sensitive and cannot be delegated"**, their Kerberos tickets can be roasted.
- **Impact**:
  - Credential harvesting for lateral movement.

#### **4. Silver Ticket Attacks (High)**
- **Attack Vector**: If an attacker compromises a service account (e.g., `sql_svc`) and obtains its NTLM hash, they can forge Kerberos tickets for any service.
- **Impact**:
  - Undetectable lateral movement.
  - Data exfiltration from databases.

#### **5. Guest Account Abuse (Medium)**
- **Attack Vector**: If the `Guest` account is enabled, attackers can use it for reconnaissance or pivoting.
- **Impact**:
  - Limited but useful for initial access.

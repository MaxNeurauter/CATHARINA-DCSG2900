### **3. Attack Path**
#### **Primary Attack Vector: Kerberoasting**
- **What it is:**
  Kerberoasting is a post-compromise attack where an attacker requests **Service Tickets (TGS)** for SPN-enabled accounts. If the attacker can brute-force or crack the **Kerberos encryption (RC4-HMAC, AES, etc.)**, they can obtain the **plaintext password hash** and crack it offline.

- **Why this is dangerous:**
  - SPNs are often assigned to **service accounts** (e.g., SQL Server, web apps), but here, a **standard user account** has an SPN, which is unusual and risky.
  - If `jon.snow` has **unconstrained delegation**, an attacker could forge **Silver Tickets** (Kerberos tickets for any service).
  - If the password is weak (e.g., "WinterIsComing123"), it can be cracked in minutes using tools like **Hashcat** or **John the Ripper**.

- **Technical Impact:**
  - **Lateral Movement:** An attacker could escalate privileges by compromising the SPN account.
  - **Golden Ticket Attacks:** If delegation is misconfigured, the attacker could impersonate any user/service in the domain.
  - **Data Theft:** Access to `CIFS/thewall.north.sevenkingdoms.local` (likely a file share) could lead to exfiltration.

#### **Secondary Attack Vectors (if applicable):**
- **AS-REP Roasting** (if the account is **pre-authentication disabled**).
- **Pass-the-Ticket** (if the attacker obtains a valid TGT).
- **Privilege Escalation** (if the SPN account has excessive permissions).

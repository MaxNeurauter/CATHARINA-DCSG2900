### **3. Attack Path (If Exploited)**
While the current settings are not catastrophically weak, **long password ages (90 days) can enable the following attack vectors:**

#### **A. Credential Stuffing & Brute-Force Attacks**
- **Impact:** If an attacker obtains a leaked password (e.g., from a third-party breach), they can reuse it for **90 days** before the victim is forced to change it.
- **Example:** A user’s password from a non-Microsoft service (e.g., LinkedIn, Gmail) is leaked. If reused in AD, the attacker has **90 days** to exploit it before the victim resets it.

#### **B. Pass-the-Hash / Golden Ticket Attacks (If Combined with Other Weaknesses)**
- **Impact:** If an attacker gains a hashed password (e.g., via **Mimikatz, DCSync, or Kerberoasting**), a **long password age means the hash remains valid for 90 days**, increasing the attack window.
- **Mitigation Impact:** Not directly exploitable alone, but **compounds risk** if other misconfigurations (e.g., **unconstrained delegation, weak SPN management**) exist.

#### **C. Insider Threats & Stale Credentials**
- **Impact:** If an employee leaves the organization, their password remains valid for **90 days**, increasing the risk of **former employees retaining access**.

### **4. Remediation**
#### **Step-by-Step Fixes:**

##### **A. Remove Overly Permissive ACLs**
1. **Remove "Everyone" Change Password Permission**
   - Navigate to **Advanced Security Settings** → Remove the **Allow "Everyone" → Change password** entry.
   - **Justification:** Prevents brute-force and password-spraying attacks.

2. **Remove ANONYMOUS LOGON Read Access**
   - Remove the **Allow "ANONYMOUS LOGON" → Read** permission on this object and descendants.
   - **Justification:** Prevents unauthenticated users from enumerating objects.

3. **Restrict Authenticated Users Permissions**
   - Replace **"Authenticated Users" → Special** with **only necessary permissions** (e.g., **Read** if required).
   - **Justification:** Reduces attack surface for lateral movement.

##### **B. Restrict Privileged Group Permissions**
4. **Limit Domain Admins & Enterprise Admins to "This Object Only"**
   - Change **Domain Admins** and **Enterprise Admins** permissions from **"This object and all descendant objects"** to **"This object only"**.
   - **Justification:** Prevents excessive privilege escalation.

5. **Remove Full Control for Non-Essential Groups**
   - Replace **Full Control** for **Key Admins** with **minimum required permissions** (e.g., **Read/Write** only where necessary).
   - **Justification:** Follows **Least Privilege Principle**.

##### **C. Secure CREATOR OWNER & ENTERPRISE DOMAIN CONTROLLERS**
6. **Remove CREATOR OWNER Full Control on Descendant Computers**
   - Remove **CREATOR OWNER → Full Control** on descendant computer objects.
   - **Justification:** Prevents unauthorized control over newly created objects.

7. **Restrict ENTERPRISE DOMAIN CONTROLLERS**
   - Replace **Full Control** with **only necessary permissions** (e.g., **Read**).
   - **Justification:** Domain Controllers should not have unnecessary write permissions.

##### **D. Disable Inheritance & Apply Explicit Deny (If Needed)**
8. **Disable Inheritance & Copy Permissions**
   - Click **Disable Inheritance** → **Convert inherited permissions into explicit permissions**.
   - **Justification:** Ensures no unintended permissions are inherited.

9. **Add Explicit Deny for Untrusted Groups (If Applicable)**
   - Add **Deny permissions** for groups that should not interact with this object (e.g., **Deny "Guest" → All**).
   - **Justification:** Explicitly blocks unauthorized access.

##### **E. Audit & Monitor Changes**
10. **Enable AD Auditing for Permission Changes**
    - Configure **Windows Event Logs** to audit **Directory Service Changes** (Event ID 5136, 5137, 5138).
    - **Justification:** Detects unauthorized permission modifications.

11. **Use PowerShell to Validate Permissions**
    ```powershell
    # Check current permissions
    Get-ACL -Path "AD:\CN=WINTERFELL,DC=north,DC=sevenkingdoms,DC=local" | Format-List

    # Remove ANONYMOUS LOGON access
    $acl = Get-ACL -Path "AD:\CN=WINTERFELL,DC=north,DC=sevenkingdoms,DC=local"
    $accessRule = $acl.Access | Where-Object { $_.IdentityReference -like "*ANONYMOUS LOGON*" }
    $acl.RemoveAccessRule($accessRule)
    Set-ACL -Path "AD:\CN=WINTERFELL,DC=north,DC=sevenkingdoms,DC=local" -AclObject $acl
    ```

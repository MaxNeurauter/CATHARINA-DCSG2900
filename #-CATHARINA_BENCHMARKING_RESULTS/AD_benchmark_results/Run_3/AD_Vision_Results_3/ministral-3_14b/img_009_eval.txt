### **3. Attack Path**
#### **Potential Attack Vectors:**
1. **Kerberoasting Attack**
   - An attacker identifies accounts with SPNs (e.g., via `SetSPN -L <user>`).
   - Requests a **TGS (Ticket-Granting Service) ticket** for the user and cracks it offline (since Kerberoasting targets service accounts, but misconfigured user delegation can also be abused).
   - If successful, the attacker gains the user’s credentials.

2. **Silver Ticket Attack**
   - If the attacker obtains the **NTLM hash** (e.g., via Mimikatz or Pass-the-Hash), they can forge a **Silver Ticket** to impersonate **any service** the user is delegated to (even if no SPN is set, unconstrained delegation allows arbitrary service impersonation).
   - This leads to **full control over any service** the attacker targets (e.g., **Domain Controller, SQL Server, Exchange**).

3. **Privilege Escalation & Lateral Movement**
   - If the user has **high privileges** (e.g., Domain Admin, Enterprise Admin), the attacker can **escalate privileges** and move laterally across the network.
   - Could lead to **Golden Ticket attacks** if the **krbtgt** account is later compromised.

#### **Technical Impact:**
- **Full domain compromise** if the account has high privileges.
- **Arbitrary code execution** on any service the user can access.
- **Persistence mechanisms** (e.g., backdooring services, creating persistent accounts).

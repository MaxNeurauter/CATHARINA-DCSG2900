### **Attack Paths & Technical Impact**

#### **1. Credential Exposure via Comments (Critically Vulnerable)**
   - **Attack Vector:** **Plaintext Password Exposure**
   - **Impact:**
     - Attackers can **directly extract passwords** (e.g., `Heartsbane` for `samwell.tarly`).
     - If `krbtgt` or `sql_svc` had passwords in comments, they could be **compromised immediately**.
   - **Likely Attack Chain:**
     - **Pass-the-Hash (PtH) / Pass-the-Ticket (PtT)** attacks.
     - **Golden Ticket / Silver Ticket attacks** if `krbtgt` is compromised.

#### **2. Kerberoasting & AS-REP Roasting (Vulnerable)**
   - **Attack Vector:** **Kerberoasting (SPN Abuse) & AS-REP Roasting (Unconstrained Delegation)**
   - **Impact:**
     - Attackers can **request TGS (Ticket-Granting Service) tickets** for service accounts (`sql_svc`, `krbtgt`).
     - If **AS-REP Roasting** is possible (users not pre-authenticated), attackers can **crack hashes offline**.
     - **Lateral movement** to **Domain Admin** if a service account has excessive privileges.
   - **Likely Attack Chain:**
     - **Mimikatz / Rubeus** → **Extract TGS tickets** → **Offline cracking (Hashcat)** → **Escalate privileges**.

#### **3. Privilege Escalation via Overly Permissive Groups (Misconfigured)**
   - **Attack Vector:** **Group Membership Abuse**
   - **Impact:**
     - If `Domain Admins` has **unnecessary members**, an attacker could **gain full domain control**.
     - **DNSAdmins**, **Key Admins**, and **Cert Publishers** are **high-risk groups** if misconfigured.
   - **Likely Attack Chain:**
     - **BloodHound** → **Identify short paths to Domain Admin** → **Escalate via group membership**.

#### **4. Lateral Movement via Service Accounts (Vulnerable)**
   - **Attack Vector:** **Pass-the-Hash / Pass-the-Ticket**
   - **Impact:**
     - If `sql_svc` or another service account has **weak credentials**, attackers can **move laterally** to other systems.
     - **SQL Server misconfigurations** (e.g., **xp_cmdshell enabled**) could lead to **full domain compromise**.

### **Remediation Steps**

#### **1. Remove Plaintext Passwords from Comments (Critical Fix)**
   - **Action:**
     - Open **Active Directory Users and Computers (ADUC)**.
     - Navigate to each user with **comments containing passwords** (e.g., `samwell.tarly`).
     - **Clear the comment field** or replace with a **generic note** (e.g., "Password managed via LAPS").
   - **PowerShell Command:**
     ```powershell
     Set-ADUser -Identity "samwell.tarly" -Description ""
     ```

#### **2. Enforce Strong Password Policies (High Priority)**
   - **Action:**
     - Open **Group Policy Management (GPMC.msc)**.
     - Navigate to:
       - **Default Domain Policy** → **Computer Configuration** → **Policies** → **Windows Settings** → **Security Settings** → **Account Policies** → **Password Policy**.
     - Enforce:
       - **Minimum password length: 14+ characters**.
       - **Password complexity: Enabled**.
       - **Password history: 24+ previous passwords**.
       - **Maximum password age: 90 days**.
       - **Account lockout: 10 failed attempts, 30-minute lockout**.

#### **3. Implement Local Admin Password Solution (LAPS) (High Priority)**
   - **Action:**
     - Install **LAPS** via:
       ```powershell
       Install-WindowsFeature -Name LAPS
       ```
     - Configure via **Group Policy**:
       - **Computer Configuration** → **Policies** → **Administrative Templates** → **LAPS**.
       - Enable **Password settings** and **Management settings**.
     - **Audit local admin passwords** and **rotate them automatically**.

#### **4. Mitigate Kerberoasting & AS-REP Roasting (High Priority)**
   - **Action:**
     - **Disable unnecessary SPNs** (Service Principal Names) for service accounts.
     - **Enable Pre-Authentication for AS-REP Roasting protection**:
       ```powershell
       Set-ADAccountControl -Identity "krbtgt" -DoesNotRequirePreAuth:$true
       ```
     - **Monitor for Kerberoasting attempts** using:
       - **Microsoft Defender for Identity**.
       - **SIEM (Splunk, ELK, Azure Sentinel)** with **Kerberoasting detection rules**.

#### **5. Restrict Domain Admin & High-Risk Group Memberships (Critical Fix)**
   - **Action:**
     - **Audit current members** of:
       - `Domain Admins`
       - `DNSAdmins`
       - `Key Admins`
       - `Cert Publishers`
       - `Enterprise Admins`
     - **Remove unnecessary accounts** (e.g., service accounts, generic users).
     - **Implement Just-In-Time (JIT) Admin access** via:
       - **Microsoft Privileged Access Management (PAM)**.
       - **BeyondTrust / CyberArk** for **temporary admin elevation**.

#### **6. Enable Account Lockout Policies (Medium Priority)**
   - **Action:**
     - In **Group Policy**:
       - **Account Policies** → **Account Lockout Policy**.
       - Set:
         - **Account lockout threshold: 10 attempts**.
         - **Reset lockout counter: 30 minutes**.
         - **Lockout duration: Until manually unlocked or 30 minutes**.

#### **7. Harden Service Accounts (Critical Fix)**
   - **Action:**
     - **Rename default service accounts** (e.g., `sql_svc` → `sql_app_svc_2024`).
     - **Use Managed Service Accounts (gMSA) or Virtual Accounts** where possible.
     - **Restrict logon rights** to only necessary machines.
     - **Disable interactive logon** for service accounts.

#### **8. Deploy Privileged Access Workstations (PAW) (Long-Term Fix)**
   - **Action:**
     - **Isolate admin workstations** from the internet.
     - **Enable BitLocker encryption**.
     - **Restrict software installation** to only approved applications.
     - **Use Microsoft Defender for Endpoint** for **advanced protection**.

#### **9. Conduct a BloodHound Assessment (Highly Recommended)**
   - **Action:**
     - Install **BloodHound** and **SharpHound**.
     - Run:
       ```powershell
       Invoke-BloodHound -CollectionMethod All
       ```
     - **Identify short paths to Domain Admin** and **remediate misconfigurations**.

#### **10. Enable Advanced Auditing (Critical for Detection)**
   - **Action:**
     - Enable **Advanced Audit Policies** in **Group Policy**:
       - **Computer Configuration** → **Policies** → **Windows Settings** → **Security Settings** → **Advanced Audit Policy Configuration**.
     - Enable:
       - **Audit Directory Service Changes** (Success & Failure).
       - **Audit Group Policy Changes**.
       - **Audit Logon Events** (4624, 4625, 4776).
       - **Audit Privilege Use** (4672).

### **3. Attack Path & Technical Impact**
#### **A. Kerberoasting (Service Ticket Abuse)**
- **Attack Vector**: Long **service ticket lifetime (10 hrs)** allows attackers to **crack hashed service tickets** offline.
- **Impact**:
  - If an attacker gains a **low-privilege account**, they can request **service tickets (e.g., for SPNs like `MSSQLSvc`)** and extract them for **offline brute-force attacks**.
  - Even if cracked slowly, a **10-hour window** increases success chances.

#### **B. Silver Ticket Attacks (Forged Service Tickets)**
- **Attack Vector**: If an attacker **compromises a domain account’s hash**, they can forge **long-lived service tickets** (due to the **10-hour max lifetime**).
- **Impact**:
  - Can **impersonate services** (e.g., `CIFS`, `LDAP`) for **persistent access**.
  - If combined with **Golden Ticket**, it leads to **full domain compromise**.

#### **C. Pass-the-Ticket (Reusing Stolen TGTs)**
- **Attack Vector**: Long **TGT lifetime (10 hrs) + renewal (7 days)** allows attackers to **reuse stolen tickets** for extended periods.
- **Impact**:
  - If an attacker **dumps LSASS** (e.g., via Mimikatz) or performs **DCSync**, they can **replay TGTs** for **privilege escalation**.
  - **7-day renewal** means even if detected later, the ticket remains valid.

#### **D. Time-Skew Attacks (Weak Clock Synchronization)**
- **Attack Vector**: If an attacker **adjusts a domain controller’s time**, they can **forge Kerberos tickets** (Golden Ticket).
- **Impact**:
  - **7-minute renegotiation window** slightly increases risk, but **5-minute tolerance** mitigates most cases.
  - Still, **tighter controls (≤ 5 minutes for both)** are recommended.

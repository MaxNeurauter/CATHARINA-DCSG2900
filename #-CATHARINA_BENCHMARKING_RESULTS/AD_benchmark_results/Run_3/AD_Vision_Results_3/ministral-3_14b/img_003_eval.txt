### **3. Attack Paths & Technical Impact**
#### **A. Kerberoasting (Most Likely Attack Vector)**
- **Target:** Service accounts (if any) or **high-privilege users** (e.g., `eddard.stark` in `Group Policy Creator Owners`).
- **How?**
  1. Attacker requests **TGS (Ticket-Granting Service) tickets** for users with **SPNs (Service Principal Names)**.
  2. If the account has a **weak password**, the attacker **cracks it offline** (e.g., using `ASREPRoast` or `Kerberoast`).
  3. If the account is in **`Domain Admins`**, the attacker gains **full domain control**.
- **Impact:**
  - **Domain compromise** (Golden Ticket attacks, lateral movement).
  - **Persistence** via GPO backdoors.

#### **B. Privilege Escalation (Group Membership Abuse)**
- **Target:** `Group Policy Creator Owners` (`eddard.stark`), `DNSAdmins`, `Domain Admins`.
- **How?**
  1. Attacker compromises a **standard user** (e.g., via phishing).
  2. If the user is in a **privileged group**, they can:
     - **Modify GPOs** → Persistence.
     - **Add themselves to `Domain Admins`** → Full control.
     - **Poison DNS records** → MitM attacks.
- **Impact:**
  - **Full domain takeover** in minutes.

#### **C. AS-REP Roasting (If `Do not require Kerberos pre-authentication` is enabled)**
- **Target:** Users with **pre-authentication disabled** (not visible here, but a common misconfiguration).
- **How?**
  1. Attacker requests **AS-REP (Authentication Service Reply)**.
  2. If pre-auth is disabled, the **hash is sent in plaintext** and cracked.
- **Impact:**
  - **Account takeover** if the user is in a privileged group.

#### **D. Golden Ticket / Silver Ticket (If `krbtgt` is compromised)**
- **Target:** `krbtgt` account (currently **not protected**).
- **How?**
  1. Attacker **steals `krbtgt` hash** (via Mimikatz, DCSync, or Kerberoasting).
  2. Forges **TGT (Ticket-Granting Ticket)** → **Golden Ticket**.
  3. Impersonates **any user**, including `Domain Admins`.
- **Impact:**
  - **Undetectable persistence** (no logs).
  - **Full domain compromise**.

#### **E. RODC Password Theft (If Misconfigured)**
- **Target:** Accounts in `Allowed RODC Password Replication Group`.
- **How?**
  1. Attacker **physically accesses an RODC** or exploits a **vulnerable RODC**.
  2. Extracts **cached passwords** (even if the account is disabled).
- **Impact:**
  - **Credential theft** if high-privilege accounts are included.

### **3. Attack Path (If Vulnerable)**
#### **A. AS-REP Roasting (Authenticated Users with Reversible Encryption)**
- **Impact:** Attackers can request **TGS (Ticket Granting Service) tickets** for users with reversible encryption, then offline-crack the password hash (e.g., using **Rubeus + Hashcat**).
- **Prerequisites:**
  - `Authenticated Users` having **"Enable per user reversibly encrypted password"** (Present in ACL).
  - User accounts with SPNs (Service Principal Names).

#### **B. Privilege Escalation (CREATOR OWNER & Authenticated Users Write Access)**
- **Impact:** An attacker could modify **computer attributes** (e.g., adding themselves to **Domain Admins** via **DCSync** or **BloodHound** path traversal).
- **Prerequisites:**
  - Write access to **computer objects** (Present via `CREATOR OWNER`).
  - Ability to create/modify objects in the OU.

#### **C. Golden Ticket / Silver Ticket (Cloneable Domain Controllers)**
- **Impact:** If an attacker compromises a **Domain Controller**, they could **clone it** and issue **Golden Tickets** (persistent domain compromise).
- **Prerequisites:**
  - Membership in **Cloneable Domain Controllers** (Present in ACL).
  - Access to **KRBTGT** account hash (via **Mimikatz** or **DCSync**).

#### **D. Unnecessary Deny Rule (Everyone: Delete Child Objects)**
- **Impact:** Could **break legitimate operations** (e.g., bulk deletions via scripts).
- **Prerequisites:**
  - Admin attempting to delete child objects in bulk.

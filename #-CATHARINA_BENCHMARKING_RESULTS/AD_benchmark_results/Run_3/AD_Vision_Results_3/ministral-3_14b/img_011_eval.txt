### **2. Assessment**
This configuration is **Critically Vulnerable** due to the following issues:

#### **A. Over-Permissive Delegation**
- **`NORTH\samwell.tarly`** and **`NORTH\vagrant`** (non-admin accounts) have **full control** over this GPO.
  - These accounts could **modify GPO settings**, including **security policies, scripts, and preferences**, which could lead to **persistent backdoors, privilege escalation, or lateral movement**.
  - If compromised, an attacker could **enable dangerous settings** (e.g., disable security auditing, enable weak authentication, or deploy malware via GPO).

#### **B. Security Filtering Misconfiguration**
- The GPO applies to **`Authenticated Users`**, meaning **every user in the domain** could be affected if malicious changes are made.
- Even though `Authenticated Users` only has **read permissions**, an attacker with **write access** (e.g., `samwell.tarly` or `vagrant`) could **modify the security filtering** to include malicious groups.

#### **C. Lack of Enforcement**
- The GPO is **not enforced**, meaning child OUs could **override** it, leading to **inconsistent security policies**.

### **3. Attack Path**
#### **Potential Attack Vectors:**
1. **GPO Abuse (Golden Ticket / Persistence)**

   - **Impact:** **Domain-wide compromise**, **lateral movement**, and **persistence**.

2. **Privilege Escalation via GPO Modification**

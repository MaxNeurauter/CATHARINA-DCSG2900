### **3. Attack Path & Technical Impact**
#### **Primary Attack Vectors:**
1. **Brute-Force & Dictionary Attacks**
   - **Why?** A **5-character minimum** with **no complexity** allows attackers to guess passwords easily (e.g., "password1", "admin", "qwerty").
   - **Impact**: Compromised credentials leading to **lateral movement, data exfiltration, or privilege escalation**.

2. **Pass-the-Hash / Credential Stuffing**
   - **Why?** Long password validity (102 years) means **stale credentials remain usable indefinitely**, increasing exposure if leaked.
   - **Impact**: Attackers reuse leaked credentials (e.g., from third-party breaches) to gain persistent access.

3. **Golden Ticket / Silver Ticket Attacks (If Kerberos is misconfigured elsewhere)**
   - **Why?** Weak passwords + long validity increase the risk of **Kerberos ticket forgery** if an attacker obtains a hash (e.g., via Mimikatz).
   - **Impact**: **Permanent domain compromise** if tickets are forged with weak credentials.

4. **Privilege Escalation (If Admin Accounts Use Weak Passwords)**
   - **Why?** High-privilege accounts (e.g., Domain Admins) with **5-character passwords** can be cracked quickly.
   - **Impact**: **Full domain takeover** (e.g., via **DCSync attacks**).

#### **Real-World Example:**
- **LockBit, Conti, and other ransomware groups** frequently exploit weak AD password policies to move laterally after initial access.
- **MITRE ATT&CK Tactic**: **Credential Access (T1003, T1558)** → **Lateral Movement (T1087)** → **Persistence (T1543)**.

---

### **4. Remediation (Step-by-Step Fix)**
#### **Immediate Actions (High Priority)**
1. **Enforce Strong Password Complexity**
   - **PowerShell Command**:
     ```powershell
     Set-ADDefaultDomainPasswordPolicy -MinPasswordLength 12 -PasswordsNeverExpire $false -ComplexityEnabled $true
     ```
   - **Alternative (GPO)**:
     - Navigate to:
       `Default Domain Policy` → `Computer Configuration` → `Policies` → `Windows Settings` → `Security Settings` → `Account Policies` → `Password Policy`
     - Set:
       - **Minimum password length**: **12+ characters**
       - **Enforce password history**: **24+ passwords**
       - **Maximum password age**: **90 days**
       - **Complexity requirements**: **Enabled**

2. **Force Password Reset for All Users**
   - **PowerShell Command**:
     ```powershell
     Get-ADUser -Filter * -Properties PasswordLastSet | Where-Object { $_.PasswordLastSet -lt (Get-Date).AddDays(-90) } | Set-ADUser -ChangePasswordAtLogon $true
     ```
   - **Impact**: Forces all users to reset passwords on next login.

3. **Audit & Disable Reversible Encryption (If Enabled Elsewhere)**
   - **PowerShell Command**:
     ```powershell
     Get-ADUser -Filter * -Properties "DoesNotRequirePreAuth" | Where-Object { $_.Enabled -eq $true } | Select-Object Name, SamAccountName
     ```
   - **Fix**: Ensure **no users/groups** have `DoesNotRequirePreAuth` (vulnerable to **AS-REP Roasting**).

#### **Long-Term Security Hardening**
4. **Implement Multi-Factor Authentication (MFA)**
   - Enforce MFA for **all privileged accounts** (Domain Admins, Enterprise Admins).
   - Use **Conditional Access Policies** in Azure AD to block legacy authentication.

5. **Enable Password Spray Defenses**
   - Deploy **Azure AD Identity Protection** or **Microsoft Defender for Identity** to detect brute-force attempts.
   - Use **GPO to lock accounts after 5 failed attempts**:
     ```powershell
     Set-ADDefaultDomainPasswordPolicy -LockoutThreshold 5 -LockoutDuration "30:00:00" -ResetLockoutCount 30
     ```

6. **Monitor for Suspicious Activity**
   - Use **SIEM tools (Splunk, Microsoft Sentinel)** to detect:
     - Multiple failed logins from unusual locations.
     - Unusual password changes (e.g., via `net user` or ADSI Edit).

7. **Educate Users on Password Hygiene**
   - Train employees on:
     - Avoiding password reuse.
     - Recognizing phishing attempts.
     - Using a **password manager** (e.g., Bitwarden, 1Password).

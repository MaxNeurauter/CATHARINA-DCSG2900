### **3. Attack Path**
#### **Likely Attack Vectors:**
1. **Brute-Force / Password Spraying Attacks**
   - Attackers can automate login attempts (e.g., using tools like **Hydra, Medusa, or Metasploit**) to guess credentials.
   - With only **5 attempts before lockout**, attackers can quickly test multiple accounts before being blocked.
   - **Impact**: Credential compromise, lateral movement, or privilege escalation.

2. **Denial-of-Service (DoS) via Account Lockouts**
   - Malicious actors can intentionally trigger lockouts to disrupt legitimate users.
   - **Impact**: Service availability issues, user frustration, and potential business downtime.

3. **Pass-the-Hash / Credential Stuffing**
   - If an attacker gains a leaked password (e.g., from a third-party breach), they can attempt it in AD before lockout.
   - **Impact**: Unauthorized access to sensitive systems.

#### **Technical Impact:**
- **Low-to-Moderate Risk**: While not critically vulnerable, this policy **lowers the bar for attackers** and increases the likelihood of successful credential attacks.
- **Compliance Violations**: Fails **NIST SP 800-63B**, **CIS Benchmarks**, and **Microsoft’s Secure Defaults** recommendations.

---

### **4. Remediation**
#### **Recommended Fixes:**
1. **Increase Account Lockout Threshold**
   - **Set to 10 or more invalid attempts** (e.g., **10–30**).
   - **Best Practice**: Use **10 for standard users**, **30+ for privileged accounts** (e.g., admins).

2. **Extend Lockout Duration**
   - **Set to 30+ minutes** (e.g., **30–60 minutes**).
   - **Best Practice**: Use **60+ minutes for high-risk environments**.

3. **Reset Counter After a Longer Period**
   - **Set to 60+ minutes** (e.g., **1 hour**).
   - **Best Practice**: Align with lockout duration to prevent rapid re-enumeration.

4. **Enable Advanced Protection (Optional but Recommended)**
   - **Enable Multi-Factor Authentication (MFA)** for all accounts.
   - **Use Conditional Access Policies** to block brute-force attempts at the network level.
   - **Log and Monitor Failed Logins** (via **Security Event ID 4776** in Windows Event Logs).

5. **Exclude Service Accounts from Lockout**
   - Ensure **service accounts** (e.g., SQL Server, scheduled tasks) are **not locked out** to prevent application failures.

### **4. Remediation: Step-by-Step Fix**
#### **Objective**:
- Remove excessive permissions.
- Restore proper inheritance.
- Block unauthorized write operations.
- Follow **least privilege principle**.

#### **Steps**:
##### **A. Remove Unnecessary Full Control Permissions**
1. **Revoke Full Control from `stannis.baratheon`**
   - Go to **Permissions** → Select **stannis.baratheon** → Click **Remove**.
   - If this is a user, assign only **Read** or **Read/Write** (if needed).

2. **Remove Full Control from `AcrossTheNarrowSea` (if not required)**
   - If this is a custom group, replace it with a **narrower group** (e.g., only **Domain Admins** if absolutely necessary).
   - If not needed, **remove the permission entirely**.

3. **Ensure Domain Admins Retain Full Control (if required for management)**
   - Keep **Domain Admins** with **Full Control**, but **do not grant to other groups unless absolutely necessary**.

##### **B. Restore Proper Inheritance**
1. Click **Disable Inheritance** → **Convert Inherited Permissions to Explicit Permissions** (if needed).
2. **Re-enable Inheritance** (if possible) and **remove conflicting permissions**.
   - Go to **Advanced** → Check **"Replace all child object permissions with inheritable permissions from this object"**.
   - Click **OK** to apply.

##### **C. Remove "Everyone" Password Change Rights**
1. Find the **Everyone** entry in the permissions list.
2. Remove the **"Change password"** right.
3. Replace with **specific groups** (e.g., **Domain Users** if password changes are needed).

##### **D. Restrict Validated Write Rights**
1. Remove **SELF** and **SFU_F** from **validated write to DNS host name/SPN**.
   - Only **Domain Admins** and **Enterprise Admins** should retain these rights.
2. If this object is a **computer account**, ensure only **Domain Admins** and **Enterprise Admins** have write access.

##### **E. Remove Unnecessary Printer/Object Creation Rights**
1. Remove **Cert Publishers** and **Print Operators** from **create/delete printer objects**.
   - These groups should **not** have object creation rights unless this is a **print server share**.

##### **F. Audit & Monitor Changes**
1. **Check Audit Policy** (via GPO or **Computer Configuration → Windows Settings → Security Settings → Advanced Audit Policy Configuration**):
   - Enable **Audit Object Access** and **Audit Permission Changes**.
2. **Monitor for Suspicious Activity**:
   - Use **Security Event ID 4663** (Special permissions granted).
   - Use **Security Event ID 4728** (Kerberos service ticket operations).

##### **H. Educate & Enforce Least Privilege**

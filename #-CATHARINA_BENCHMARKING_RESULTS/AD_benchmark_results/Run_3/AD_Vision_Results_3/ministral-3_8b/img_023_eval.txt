**3. Attack Path (if applicable):**

#### **Misconfigured Policy:**
- **"Force logoff when logon hours expire" (Disabled)**
  - **Attack Vector:** **Session Hijacking / Persistent Access**
  - **Technical Impact:**
    - If logon hours expire, users are **not automatically logged off**, allowing attackers to maintain unauthorized sessions.
    - An attacker who gains access during valid logon hours could **remain active** even after the legitimate user’s session should expire, increasing the window for lateral movement or data exfiltration.
    - This can also enable **credential stuffing** if an attacker uses stolen credentials during valid hours and avoids detection by lingering after hours.

---

**4. Remediation:**

#### **Step-by-Step Fix for "Force logoff when logon hours expire":**
1. **Open Group Policy Management Console (GPMC):**
   - Press `Win + R`, type `gpmc.msc`, and press **Enter**.

2. **Navigate to the Relevant GPO:**
   - Expand **Forest > Domains > [Your Domain] > Group Policy Objects**.
   - Locate the GPO linked to the affected OUs/users (e.g., "Default Domain Policy" or a custom policy).
   - Right-click the policy and select **Edit**.

3. **Enable the Policy:**
   - Navigate to:
     **Computer Configuration > Policies > Windows Settings > Security Settings > Local Policies > Security Options**.
   - Find the policy:
     **"Interactive logon: Machine inactivity limit"** (if available) **OR**
     **"Force logoff when logon hours expire"** (if present in your environment).
   - If the exact policy is missing, use the **closest alternative**:
     - Enable **"Interactive logon: Machine inactivity limit"** and set it to a reasonable value (e.g., **1 hour**).
     - Alternatively, enforce **logon hours** via **Active Directory Users and Computers** (ADUC) and ensure the policy is enforced.

4. **Enforce via ADUC (if GPO is unavailable):**
   - Open **Active Directory Users and Computers** (`dsa.msc`).
   - Right-click the **OU containing the users** > **Properties** > **Logon Workstations** tab.
   - Set **Logon hours** to restrict access to specific times.
   - Ensure **"Force logoff"** is enabled in the **Security Options** GPO (if available) or use **Group Policy Preferences** to enforce logoff scripts.

5. **Test the Configuration:**
   - Log in during valid hours and verify the system enforces logoff after hours expire.
   - Check **Event Viewer** (`eventvwr.msc`) under **Security Log** for events related to logoff enforcement (Event ID 4649 for logoff).

6. **Monitor and Audit:**
   - Deploy **audit policies** to track logon/logoff events (e.g., **Audit Logon Events** in **Local Security Policy**).
   - Use **Advanced Audit Policies** (e.g., **"Audit Other Logon/Logoff Events"**) to detect unauthorized sessions.

---

- **"Allow anonymous SID/Name translation"** is **securely disabled**, reducing **SID enumeration attacks** (e.g., identifying users/groups via null sessions).

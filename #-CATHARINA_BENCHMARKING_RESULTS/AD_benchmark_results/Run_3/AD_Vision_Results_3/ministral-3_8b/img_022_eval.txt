| **Policy**                          | **Setting**                          |

| Minimum password age                | 1 day                                |
| Minimum password length             | 5 characters                         |
| Password must meet complexity reqs. | Disabled                             |

---

### **2. Assessment**
**Critically Vulnerable**
This configuration contains **multiple severe security flaws** that significantly weaken AD security.

---

### **3. Attack Path & Technical Impact**
The following attack vectors are enabled or exacerbated by these settings:

#### **A. Password Expiration Bypass (Golden Ticket / AS-REP Roasting)**
- **Issue:** `Maximum password age = 37,201 days` (effectively **never expires**).
  - Allows attackers to **capture a user’s hash (via tools like Mimikatz, BloodHound, or DumpSec)** and create **Golden Tickets** or **Silver Tickets** without password changes.
  - Enables **persistent lateral movement** without detection.

#### **B. Weak Password Complexity (Brute Force / Credential Stuffing)**
- **Issue:** `Password must meet complexity requirements = Disabled` + `Minimum password length = 5 characters`.
  - Attackers can use **short, simple passwords** (e.g., `password123`, `admin`, `qwerty`).
  - Enables **offline brute-force attacks** (e.g., via **Hashcat** or **John the Ripper**) on stolen hashes.
  - **AS-REP Roasting** becomes trivial if passwords are weak (e.g., `1 day minimum age` + no complexity).

#### **C. Password Reuse (Credential Theft Exploitation)**
- **Issue:** `Enforce password history = 24 passwords remembered`.
  - While this prevents immediate reuse, attackers can **replay old passwords** if they have access to historical hashes (e.g., from backups or shadow credentials).
  - If an attacker steals a password, they can **revert to an old password** if it was reused.

#### **D. Reversible Encryption Risk (If Enabled Elsewhere)**
- **Issue:** `Store passwords using reversible encryption = Disabled` (currently safe, but if enabled, it would allow **plaintext password extraction** from the AD database).

#### **E. Kerberoasting (Mitigated but Not Prevented)**
- While not directly enabled by these settings, weak passwords + **no complexity** make Kerberoasting more effective (attackers can crack service account passwords faster).

---
### **4. Remediation: Step-by-Step Fix**
#### **Immediate Actions (Critical)**
1. **Set a Realistic Password Expiration Policy**
   - **Change `Maximum password age` to 90 days** (or as per organizational policy).
   - **Justification:** Prevents Golden Ticket attacks by forcing periodic password changes.

2. **Enforce Strong Password Complexity**
   - **Enable `Password must meet complexity requirements`** (check the box).
   - **Set `Minimum password length` to 12+ characters** (minimum industry standard).
   - **Justification:** Blocks brute-force attacks and weak credentials.

3. **Adjust Password History**
   - **Keep `Enforce password history = 24 passwords remembered`** (or higher if possible).
   - **Justification:** Prevents immediate password reuse without being overly restrictive.

4. **Disable Reversible Encryption (If Applicable Elsewhere)**
   - Ensure `Store passwords using reversible encryption` is **disabled** in all GPOs.
   - **Justification:** Prevents plaintext password extraction from AD databases.

5. **Audit and Rotate Compromised Credentials**
   - Use **BloodHound** or **ADAuditPlus** to detect stale passwords or suspicious access.
   - **Force password changes** for all users with passwords older than 90 days.

#### **Long-Term Security Hardening**
- **Enable Multi-Factor Authentication (MFA)** for all AD accounts (especially admins).
- **Implement Least Privilege Access** (restrict `Domain Admin` rights).
- **Monitor for Golden Ticket/Silver Ticket attacks** using tools like **ADAudit, CrowdStrike, or Microsoft Defender for Identity**.
- **Enable Password Protection Policies** (e.g., block common passwords, enforce dictionary checks).
- **Consider Passwordless Authentication** (e.g., FIDO2, TOTP) for high-risk accounts.

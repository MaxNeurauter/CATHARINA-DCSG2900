### **4. Remediation: Step-by-Step Fixes**
#### **Fix 1: Remove Unnecessary Password Reset Permissions**
**Goal**: Ensure only authorized admins can reset passwords.
**Steps**:
1. Open **Active Directory Users and Computers**.
2. Locate `jaime.lannister` and open **Properties** > **Delegation** tab.
3. Remove `tywin.lannister` from the **"Reset password"** permission list.
4. If `tywin.lannister` is not an admin, revoke their access entirely.
5. **Best Practice**: Use **Password Reset Groups** (e.g., `Password Reset Admins`) instead of granting to individual users.

#### **Fix 2: Restrict `Authenticated Users` Permissions**
**Goal**: Limit read permissions to only necessary data.
**Steps**:
1. In **Advanced Features** view, go to **jaime.lannister** > **Properties** > **Security** tab.
2. Remove `Authenticated Users` from:
   - **Read personal information**
   - **Read web information** (if not required)
3. Keep only **Read general information** (standard for all users).
4. **Best Practice**: Use **Access Control Lists (ACLs)** to restrict sensitive attributes (e.g., `msExchMailboxGUID`, `proxyAddresses`).

#### **Fix 3: Disable Inheritance for Legacy Permissions**
**Goal**: Prevent unintended access via `Pre-Windows 2000 Compatible Access`.
**Steps**:
1. In the **Security** tab for `jaime.lannister`, click **Advanced**.

3. Click **OK** and apply changes.

#### **Fix 4: Audit and Remove Unused Groups**

#### **Fix 5: Enforce Least Privilege for Domain Admins**
**Goal**: Ensure `Domain Admins` are the only ones with Full Control.
**Steps**:
1. Verify no other groups/users have **Full Control** on `jaime.lannister`.
2. If needed, revoke permissions from unexpected principals (e.g., `Account Operators` if they shouldn’t manage this user).

#### **Fix 6: Enable Password Protection Policies**
**Goal**: Prevent password resets for locked-out accounts.
**Steps**:
1. In **Group Policy Management**, navigate to:
   `Default Domain Policy` > **Computer Configuration** > **Policies** > **Windows Settings** > **Security Settings** > **Account Policies** > **Password Policy**.

   - **Minimum password length**: 12+ characters.
   - **Password must meet complexity requirements**: Enabled.
3. Enable **Account Lockout Policy**:

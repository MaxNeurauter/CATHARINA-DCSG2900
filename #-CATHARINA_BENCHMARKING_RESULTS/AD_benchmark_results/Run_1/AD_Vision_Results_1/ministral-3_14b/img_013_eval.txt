### **2. Assessment**
#### **Misconfigured & Critically Vulnerable Findings:**
✅ **Secure:**
- Enterprise Admins and Domain Admins having **Full Control** is expected.
- Replication rights for **ENTERPRISE DOMAIN CONTROLLERS** are standard.

⚠️ **Misconfigured:**
- **`Authenticated Users`** having **"Enable per-user reversible encryption"** is **weak security practice** (reversible encryption allows password cracking).
- **`Cloneable Domain Controllers`** having **"Allow a DC to create a clone"** is **risky** if not strictly controlled (could aid in Golden Ticket attacks).

❌ **Critically Vulnerable:**
- **`ANONYMOUS LOGON`** having **Read** access on **this object + all descendants** is **extremely dangerous** in AD. Attackers can:
  - Enumerate sensitive objects (users, groups, GPOs).
  - Discover SPNs for **AS-REP Roasting** or **Kerberoasting**.
  - Map the AD structure for further attacks.

---

### **3. Attack Path**
#### **Primary Attack Vectors:**
| **Vulnerability**                     | **Attack Type**                     | **Technical Impact**                                                                 |
|----------------------------------------|-------------------------------------|--------------------------------------------------------------------------------------|
| **Anonymous Read Access**              | **AD Reconnaissance**               | Attackers can enumerate all objects, discover SPNs, and prepare for roasting attacks. |
|                                        | **AS-REP Roasting**                 | If `Do not require Kerberos pre-authentication` is enabled on accounts, AS-REP roasting can extract hashes. |
|                                        | **Kerberoasting**                   | If Service Principal Names (SPNs) are found, attackers can request TGS tickets and crack them offline. |
| **Reversible Encryption**              | **Password Cracking**               | If passwords are stored with reversible encryption, they can be extracted and cracked. |
| **Cloneable Domain Controller Rights** | **Golden Ticket Attack**            | If an attacker gains a cloneable DC right, they could forge a Domain Controller and issue Golden Tickets. |

#### **Likely Attack Chain:**
1. **Anonymous user enumerates AD** → Discovers sensitive SPNs.
2. **Performs AS-REP Roasting** → Extracts hashes of accounts without pre-authentication.
3. **Cracks hashes offline** → Gains a valid account (e.g., a service account with SPN).
4. **Performs Kerberoasting** → Extracts and cracks TGS tickets for higher privileges.
5. **Abuses `Cloneable Domain Controller` rights** → Creates a rogue DC and issues Golden Tickets for full domain compromise.

---

### **4. Remediation (Step-by-Step Fixes)**

#### **🔹 Fix 1: Remove Anonymous Read Access (Critical)**
1. **Open Active Directory Users and Computers (ADUC)**.
2. Navigate to the **OU/Object** with misconfigured permissions.
3. Right-click → **Properties** → **Security** → **Advanced**.
4. Remove the **`ANONYMOUS LOGON`** entry with **Read** permissions.
5. **Apply** changes and verify with:
   ```powershell
   Get-ACL -Path "AD:\OU=SensitiveOU,DC=domain,DC=com" | Select-Object -ExpandProperty Access | Where-Object { $_.IdentityReference -like "*ANONYMOUS*" }
   ```

#### **🔹 Fix 2: Disable Reversible Encryption for Authenticated Users**
1. Open **Group Policy Management (GPMC.msc)**.
2. Navigate to the **Default Domain Policy** or relevant GPO.
3. Go to:
   **Computer Configuration → Windows Settings → Security Settings → Account Policies → Kerberos Policy**
4. Set:
   - **Enforce user logon restrictions** → **Enabled**
   - **Maximum ticket age** → **Appropriate value (e.g., 10 hours)**
5. Under **Domain Controller Policy**:
   - **Disable "Store passwords using reversible encryption"** (if enabled).
6. Run:
   ```powershell
   auditpol /set /subcategory:"Kerberos Authentication Service" /success:enable /failure:enable
   ```

#### **🔹 Fix 3: Restrict Cloneable Domain Controller Rights**
1. Open **ADUC** → Navigate to the **Domain Controllers OU**.
2. Right-click → **Properties** → **Security** → **Advanced**.
3. Remove **"Cloneable Domain Controllers"** group from:
   - **"Allow a DC to create a clone"** (if present).
4. Alternatively, **remove the group entirely** if not needed:
   ```powershell
   Remove-ADGroup -Identity "Cloneable Domain Controllers" -Confirm:$false
   ```
   *(Note: This may break some recovery scenarios; consult Microsoft docs before deletion.)*

#### **🔹 Fix 4: Enable Protective Measures Against Roasting**
1. **Enable Kerberos Pre-Authentication for All Accounts**:
   ```powershell
   Set-ADAccountControl -Identity "Domain Users" -DoesNotRequirePreAuth:$true
   ```
   *(Actually, the correct command to **require** pre-auth is:)*
   ```powershell
   Set-ADAccountControl -Identity "Domain Users" -DoesNotRequirePreAuth:$false
   ```
2. **Audit SPN Misconfigurations**:
   ```powershell
   Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName | Select-Object Name,ServicePrincipalName
   ```
3. **Remove Unused SPNs**:
   ```powershell
   Set-ADUser -Identity "UserWithSPN" -ServicePrincipalNames @{Remove="HOST/service.example.com"}
   ```

#### **🔹 Fix 5: Implement Least Privilege for Key Admins**
1. **Audit Key Admins Group**:
   ```powershell
   Get-ADGroupMember -Identity "Key Admins" | Select-Object Name
   ```
2. **Remove unnecessary members** and restrict their permissions.

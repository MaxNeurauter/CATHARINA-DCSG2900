### **3. Attack Path**
#### **A. DNS Hijacking (Most Critical)**
- **Attack Vector:** **DNS Spoofing / Cache Poisoning**
- **Technical Impact:**
  - An attacker with **`Everyone` write access** to the **DNS host name** can modify DNS records, redirecting traffic to malicious servers.
  - This enables **phishing, man-in-the-middle (MITM) attacks, and lateral movement** within the domain.
  - Example: Changing `golden.tgt` (a domain controller) to point to an attacker-controlled server for **Silver Ticket attacks**.

#### **B. Privilege Escalation**
- **Attack Vector:** **Abuse of `Create/Delete all child objects`**
  - A standard user (`stannis.baratheon`) can **create malicious child objects** (e.g., fake admin accounts, GPOs, or scheduled tasks).
  - Example: Creating a **stale computer account** for **AS-REP Roasting** or a **fake admin account** for lateral movement.

#### **C. Kerberoasting / AS-REP Roasting**
- **Attack Vector:** **Excessive `List` permissions on descendant objects**
  - If `Authenticated Users` can **enumerate SPNs (Service Principal Names)**, an attacker can perform **Kerberoasting** to crack service account passwords.
  - If **`Pre-Windows 2000` compatible access** allows **unconstrained delegation**, **AS-REP Roasting** becomes possible.

#### **D. Golden Ticket / Silver Ticket Attacks**
- **Attack Vector:** **`Validated write to service principal name` + `SELF` permissions**
  - If an attacker gains a **service account ticket**, they can modify SPNs and create **Golden/Silver Tickets** for persistent domain compromise.

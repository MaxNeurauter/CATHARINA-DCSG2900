### **3. Attack Path (If Exploitable)**
While the current policy is **not critically weak**, the following **potential attack vectors** could still be leveraged if combined with other misconfigurations:

#### **A. Brute-Force / Spraying Attacks (Reduced but Possible)**
- **Impact:** If an attacker gains access to password hashes (e.g., via **Pass-the-Hash** or **Mimikatz**), a **90-day password validity window** increases the chance of successful brute-forcing.
- **Mitigation:** Shorter password expiration (e.g., **60 days**) reduces attack surface.

#### **B. Credential Stuffing / Password Reuse**
- **Impact:** If users reuse passwords across systems, a **10-password history** may not prevent reuse if an old password is leaked.
- **Mitigation:** Enforce **longer password history** (e.g., **24 passwords**) or **block common passwords**.

#### **C. Privilege Escalation via Stale Passwords**
- **Impact:** If an admin account’s password is **90 days old**, an attacker with initial access (e.g., via **RDP brute-force**) could exploit it before expiration.
- **Mitigation:** Apply **different policies for admins** (e.g., **30-day max password age**).

#### **D. Lack of MFA (Biggest Risk)**
- **Impact:** Even with strong passwords, **MFA is the #1 defense** against credential theft (e.g., **Golden Ticket, Silver Ticket, or stolen hashes**).
- **Mitigation:** **Enforce MFA for all users**, especially admins.

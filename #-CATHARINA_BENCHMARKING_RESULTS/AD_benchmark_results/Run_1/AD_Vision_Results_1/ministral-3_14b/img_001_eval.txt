### **3. Attack Path (If Vulnerable)**
#### **Primary Risk: LDAP Signing Disabled**
- **Attack Vector:** **LDAP Signing/Integrity Attacks (CVE-2020-17049, MITM, Credential Theft)**
- **Technical Impact:**
  - An attacker on the network can **intercept, modify, or forge LDAP traffic** (e.g., password changes, user modifications).
  - **No integrity checks** → **Pass-the-Hash (PtH) attacks** become easier.
  - **Kerberoasting & AS-REP Roasting** attacks may be facilitated if combined with other misconfigurations.
  - **Silver Ticket attacks** could be more effective if an attacker compromises a domain account (since LDAP responses can be spoofed).

#### **Secondary Risk: Potential Privilege Escalation**
- If an attacker gains a **low-privilege account**, they could **relay signed (but unencrypted) LDAP requests** to escalate privileges via **misconfigured delegation**.

---

### **4. Remediation (Step-by-Step Fix)**
#### **Critical Fix: Enable LDAP Signing & Encryption**
1. **Open Group Policy Management Console (GPMC):**
   - Run `gpmc.msc` on a **Domain Controller**.
2. **Navigate to:**
   - **Computer Configuration** → **Windows Settings** → **Security Settings** → **Local Policies** → **Security Options**.
3. **Modify the following settings:**
   - **Domain Controller: LDAP server signing requirements**
     - **Set to:** **Require signing** (or **Require signing and sealing** for stronger protection).
     - *Location:*
       - **Domain-wide via GPO:**
         `Computer Configuration → Policies → Windows Settings → Security Settings → System Services → LDAP Server → Properties → Signing Requirements → Require Signing`
       - **Or via Registry (if GPO unavailable):**
         `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NTDS\Parameters`
         Add **DWORD** `LDAPServerIntegrity` = `1` (Require Signing).
4. **Force Group Policy Update:**
   - Run on **all Domain Controllers**:
     ```powershell
     gpupdate /force
     ```
5. **Verify LDAP Signing is Enforced:**
   - Use **PortQry** or **Wireshark** to confirm LDAP traffic is signed.
   - Test with:
     ```powershell
     (Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\NTDS\Parameters").LDAPServerIntegrity
     ```
     (Should return `1` or `2` for signing/sealing.)

#### **Additional Hardening (Recommended)**
- **Enable LDAPS (LDAP over TLS):**
  - Bind LDAP to **TCP 636** and configure **SSL certificates**.
  - Use **PowerShell**:
    ```powershell
    Install-WindowsFeature -Name RSAT-AD-Tools
    Set-ADDomainController -Identity <DC_Name> -LDAPSSupported $true
    ```
- **Enforce SMB Signing (Already Enabled in Screenshot)**

#### **Immediate Mitigation (If Attack Detected)**
- **Isolate Domain Controllers** from untrusted networks.
- **Rotate all domain account passwords** if LDAP signing was disabled for an extended period.
- **Deploy Network Monitoring** (e.g., **Microsoft Sentinel, Zeek/Bro**) to detect unusual LDAP traffic.

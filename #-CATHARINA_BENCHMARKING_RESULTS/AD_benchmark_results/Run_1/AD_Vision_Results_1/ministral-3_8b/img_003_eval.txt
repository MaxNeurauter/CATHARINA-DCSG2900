### **4. Remediation Steps**
#### **A. Immediate Fixes**
1. **Disable the Guest Account**:
   - Right-click `Guest` → Properties → Uncheck "Account is disabled" (if enabled) and remove from all groups.

2. **Enforce Password Policies**:
   - Open **Group Policy Management** → Create a new GPO for password policies.
   - Set:
     - Minimum password length: **12+ characters**.
     - Enforce password history: **24 entries**.
     - Password never expires: **Unchecked** (except for service accounts).
     - Account lockout threshold: **5 failed attempts**.
     - Lockout duration: **30 minutes**.

3. **Create "Protected Users" Group**:
   - Add `krbtgt` and other sensitive accounts to the "Protected Users" group.
   - Apply the "Protected Users" GPO (enforces Kerberos armoring and AS-REP requirements).

4. **Restrict Domain Admins**:
   - Remove all non-essential users from `Domain Admins`.
   - Use **Restricted Groups** GPO to ensure Domain Admins are **not** local admins on member servers.

5. **Restrict DnsAdmins and Cert Publishers**:
   - Limit membership to only necessary accounts.
   - Apply least privilege (e.g., only allow DNS/Certificate management, not domain admin rights).

6. **Review Password Replication**:
   - Ensure only necessary accounts are in "Allowed RODC Password Replication."
   - Exclude sensitive accounts from "Denied RODC Password Replication."

#### **B. Long-Term Fixes**
1. **Enable "Protected Users" GPO**:
   - This prevents AS-REP Roasting and enforces Kerberos armoring.
   - Apply to all sensitive accounts (e.g., `krbtgt`, service accounts).

2. **Implement Least Privilege**:
   - Use **Role-Based Access Control (RBAC)** for all groups (e.g., `DnsAdmins`, `Cert Publishers`).
   - Avoid giving broad permissions to users like `brandon.stark` or `arya.stark` unless necessary.

3. **Monitor for Kerberoasting**:
   - Use tools like **BloodHound** or **ADAudit Plus** to detect vulnerable service accounts.
   - Rotate passwords for accounts targeted by Kerberoasting.

4. **Enable Advanced Auditing**:
   - Configure **Security Event Log** to audit:
     - Logon events (4624, 4625, 4634).
     - Privilege use (4672).
     - Directory access (4662, 4663).

5. **Disable Cloneable Domain Controllers**:
   - Ensure no unnecessary domain controllers are cloneable.

6. **Regular Audits**:
   - Use **Microsoft Security Compliance Toolkit** or **CIS Benchmarks** for AD hardening.
   - Schedule quarterly reviews of:
     - Group memberships.
     - Password policies.

### **3. Attack Paths and Technical Impact**
#### **A. AS-REP Roasting (Kerberos Authentication Attack)**
- **Vector**: The `Protected Users` group is empty, meaning all users (including fictional ones) are vulnerable to AS-REP Roasting.
- **Technical Impact**:
  - Attackers can request AS-REP tokens for accounts (even if passwords are unknown) and crack them offline.
  - If successful, attackers gain access to the user’s session, including privileged accounts if fictional users are in `Domain Admins`.
- **Tools**: Hashcat, Impacket’s `GetNPUsers`.

#### **B. Privilege Escalation via RODC Password Replication**
- **Vector**: Fictional users (e.g., `arya.stark`) are in the `Allowed RODC Password Replication Group`.
  - If an attacker compromises an RODC, they can extract hashed passwords of these users.
  - If these users are also in `Domain Admins` or other sensitive groups, the attacker can escalate privileges.
- **Technical Impact**:
  - Compromise of domain-wide credentials.
  - Lateral movement to other systems.

#### **C. Brute-Force/Dictionary Attacks**
- **Vector**: Passwords tied to public knowledge (e.g., "Heartsbane" for `samwell.tarly`).
- **Technical Impact**:
  - Attackers can guess passwords easily and gain access to user accounts.
  - If the user is in `Domain Admins`, the attacker gains full control of the domain.

#### **D. Certificate Fraud (if `Cert Publishers` is compromised)**
- **Vector**: If fictional users are added to `Cert Publishers`, they could issue fraudulent certificates for other users or services.
- **Technical Impact**:
  - Impersonation of other users or services (e.g., golden tickets, silver tickets).

### **3. Attack Path & Technical Impact**
#### **A. "Access this computer from the network" (Everyone)**
- **Attack Vector:** **Lateral Movement / Unauthorized Remote Access**
- **Impact:**
  - Any authenticated user (including domain users, guests, or attackers) can connect to the system over the network.
  - Enables **pass-the-hash attacks**, **man-in-the-middle (MITM) attacks**, and **brute-force credential guessing**.
  - If combined with weak passwords, an attacker could gain a foothold in the network.

#### **B. "Bypass traverse checking" (Everyone, Authenticated Users)**
- **Attack Vector:** **Directory Traversal / Privilege Escalation**
- **Impact:**
  - Allows attackers to navigate outside restricted directories (e.g., `C:\Windows\System32`).
  - Can lead to **unauthorized access to sensitive files** (e.g., `NTDS.dit`, `sam.db`).
  - Enables **exfiltration of credentials** or **execution of malicious scripts** in restricted locations.

#### **C. "Shut down the system" (Print Operators, Server Operators, Backup Operators)**
- **Attack Vector:** **Denial of Service (DoS) / Privilege Escalation**
- **Impact:**
  - Non-admin groups should **not** be able to shut down servers, as this could disrupt operations.
  - If an attacker gains access to a `Server Operator` account, they could **crash critical systems**.

#### **D. "Log on as a batch job" (Backup Operators)**
- **Attack Vector:** **Privilege Escalation via Scheduled Tasks**
- **Impact:**
  - Backup Operators should **not** have the ability to run arbitrary batch jobs, as this could allow **malicious script execution** with elevated privileges.

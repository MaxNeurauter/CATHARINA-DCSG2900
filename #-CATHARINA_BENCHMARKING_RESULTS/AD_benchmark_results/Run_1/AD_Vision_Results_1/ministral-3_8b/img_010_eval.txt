### **2. Assessment**
#### **Security Concerns Identified:**
- **Service Principal Name (SPN) on a User Account**:
  - SPNs are typically assigned to **service accounts** (e.g., computer accounts or dedicated service accounts) to enable Kerberos authentication for services like SQL, RDP, or file sharing.

### **3. Attack Path**
#### **Primary Attack Vector: Kerberoasting**
- **Technical Impact**:
  - Attackers can enumerate all SPNs in the domain (using tools like `Get-NetUser` or `Rubeus`) and extract Kerberos tickets (hashes) associated with them.
  - These hashes can be cracked offline (e.g., using Hashcat or John the Ripper) to obtain plaintext passwords, even if the account is not sensitive.
  - If the user account has elevated privileges (e.g., part of a privileged group), the stolen credentials could be used for privilege escalation.

- **Example Attack Flow**:
  1. Attacker enumerates SPNs in the domain.
  2. Requests Kerberos tickets for SPNs (e.g., `kerberos.exe get /target:thewall.north.sevenkingdoms.local`).
  3. Cracks the extracted hashes to obtain the password.

### 4. Weaknesses & Risks (Cons)  
- **Critical Single Points of Failure (SPOFs)**:  
  - The **2960 switch** is a SPOF: Failure of this switch disrupts *all* endpoints.  
  - The **2901 router** is a SPOF: Failure severs access to the external "Subnet."  
- **No segmentation**:  
  - All 5 PCs share a single broadcast domain. This risks broadcast storms, network congestion, and security breaches (e.g., a compromised PC can access all others).  
  - No VLANs, firewalls, or security boundaries exist between endpoints or the external "Subnet."  
- **Inefficient design**:  
  - The flat topology violates the **Cisco 3-Tier Hierarchical Model** (no access/distribution/core layers). The router is overburdened as both a gateway *and* inter-VLAN router (if future segmentation is added).  
  - The "Subnet" block is abstracted without context, implying a security gap (e.g., no firewall between internal/external traffic).  
- **Lack of security**:  
  - No firewall/ASA to protect against external threats. The router alone cannot replace a dedicated security appliance.  

---

### 5. Recommendations  
1. **Implement VLANs and Layer 3 Segmentation**:  
   - Create VLANs (e.g., VLAN 10 for PCs, VLAN 20 for future servers) on the 2960 switch. Use the router (or replace it with a layer 3 switch like a 3560) for inter-VLAN routing. This isolates traffic, reduces broadcast domains, and improves security.  
   - *Why*: Aligns with the Cisco 3-Tier Model (access layer = switch, distribution layer = layer 3 switch/router).  

2. **Add Redundancy for SPOFs**:  
   - **Switch redundancy**: Deploy a second 2960 switch (or stackable switch) with STP (Spanning Tree Protocol) to prevent loops. Add a second uplink to the router (e.g., using HSRP for failover).  
   - **Router redundancy**: Use a second 2901 router with HSRP (or VRRP) for gateway failover.  
   - *Why*: Eliminates the 2960 and 2901 as SPOFs, ensuring uptime for critical services.  

3. **Introduce a Firewall for Security Boundaries**:  
   - Place a dedicated firewall (e.g., Cisco ASA) between the router and the "Subnet." Isolate the internal network (VLANs) from the external "Subnet" via the firewall.  
   - *Why*: Enforces security policies, prevents external threats, and supports DMZ segmentation if needed (e.g., for servers).

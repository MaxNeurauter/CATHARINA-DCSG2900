### 4. Weaknesses & Risks (Cons)
#### **A. Lack of Segmentation**
- **No VLANs:** All devices share the same broadcast domain, which can lead to broadcast storms, inefficient traffic, and security risks (e.g., a compromised PC can impact others).
- **No Subnet Isolation:** No logical separation between PCs or between the local network and the external subnet.

#### **B. Single Points of Failure (SPOF)**
- **Switch:** If the 2960 switch fails, the entire local network goes down.
- **Router:** If the 2901 router fails, connectivity to the external subnet is lost.
- **No Redundancy:** There are no backup paths or redundant components (e.g., no secondary switch or failover router).

#### **C. Security Risks**
- **No Firewall or Access Control:** The router is directly connected to the switch without any security boundaries (e.g., firewall, ACLs) between the local network and the external subnet.
- **Flat Network:** Easy for malware or unauthorized traffic to spread across all PCs.

#### **D. Scalability Issues**
- The design lacks scalability for growth (e.g., adding more PCs or departments would require additional segmentation).

---

### 5. Recommendations
#### **1. Implement VLAN Segmentation**
   - **Action:** Divide the network into VLANs (e.g., VLAN for PCs, VLAN for servers, VLAN for VoIP) to isolate traffic and reduce broadcast domains.
   - **Why:** Improves security, performance, and manageability by logically separating devices.
   - **Example:** Use VLAN 10 for PCs, VLAN 20 for a future server segment, and configure a trunk link between switches if expanding.

#### **2. Add Redundancy and High Availability**
   - **Action:**
     - Deploy a **second switch** (e.g., Cisco 2960 or similar) and use **LACP (Link Aggregation Control Protocol)** or **STP (Spanning Tree Protocol)** for redundancy.
     - If budget allows, add a **redundant router** (e.g., HSRP or VRRP for failover).
   - **Why:** Eliminates SPOFs and ensures uptime during hardware failures.

#### **3. Introduce Security Boundaries**
   - **Action:**
     - Place a **firewall** (e.g., Cisco ASA or Palo Alto) between the router and the switch to filter traffic between the local network and the external subnet.
     - Configure **Access Control Lists (ACLs)** on the router to restrict unauthorized access.
   - **Why:** Protects against unauthorized access, malware, and external threats.

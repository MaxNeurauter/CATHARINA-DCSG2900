#### **3. Strengths (Pros)**  
- **Simplicity**: Easy to manage for small-scale deployments (e.g., small offices with ≤10 devices).  
- **Physical Connectivity**: All devices are physically connected via cabling, ensuring basic functionality.  
- **Layer 2 Switching**: Switches enable efficient local traffic forwarding within segments.  

---

#### **4. Weaknesses & Risks (Cons)**  
- **Single Points of Failure (SPOF)**:  
  - **Router0**: Critical for internet access. Failure disables all external connectivity.  
  - **Switch0**: SPOF for the entire network (links Router0 and Switch1). Failure isolates PCs from the rest of the network.  
  - **Switch1**: SPOF for the server segment. Failure splits the network into two isolated parts (PCs vs. servers/laptops/printers).  
  - **Switch2**: SPOF for the right-side segment (laptops/printers/AP). Failure disrupts wireless and print devices.  
- **Inefficient Traffic Flow**:  
  - All devices share a single broadcast domain → **broadcast storms** (e.g., a PC broadcast floods all devices).  
  - No traffic isolation → **security risks** (e.g., a compromised PC can access servers/prints without restriction).  
- **No Security Boundaries**:  
  - No firewall between internal devices and the internet.  
  - Wireless (AP) and wired devices share the same segment → **unauthorized access** risks.  
- **Lack of Redundancy**:  
  - No redundant paths for internet access or switch links.  
  - No high-availability (HA) pairs for routers/switches.  

---

#### **5. Recommendations**  
1. **Implement VLAN Segmentation**:  
   - Create separate VLANs for:  
     - PCs (VLAN10)  
     - Servers (VLAN20)  
     - Laptops (VLAN30)  
     - Printers (VLAN40)  
     - Wireless (VLAN50)  
   - Use a **Layer 3 switch** (e.g., Cisco 3560) to route between VLANs, reducing broadcast domains and improving security.  

2. **Add Redundancy**:  
   - Deploy a **second router** in an HA pair (e.g., failover via HSRP/VRRP) for internet access.  
   - Install **multiple switches** in parallel (e.g., redundant links between Switch0 and Switch1 to avoid SPOF).  

3. **Introduce Security Layers**:  
   - Place a **firewall** (e.g., Cisco ASA) between the internal network and the internet.  
   - Segment the wireless network into a dedicated VLAN (VLAN50) with **WPA3 encryption** and **ACLs** to restrict access.  

---

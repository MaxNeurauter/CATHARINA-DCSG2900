### **4. Weaknesses & Risks (Cons)**  
- **Critical SPOFs & Security Gaps:**  
  - **No Firewall for DMZ:** Servers (Server1–2, DHCP-Server) are **directly exposed** to the internal network *and* Internet (no firewall between core and servers). The DHCP server is also on the same segment as servers, risking **malicious DHCP spoofing**.  
  - **Single DHCP Server:** A single point of failure (SPOF) for IP address allocation; no high-availability (HA) pairing.  
- **Inefficient Design & Risky Configuration:**  
  - **Lack of VLAN Segmentation:** All traffic (PCs, phones, servers) shares the same Layer 2 domain (no VLANs). This risks broadcast storms, ARP spoofing, and unauthorized access (e.g., PC0 could scan servers).  
  - **Unnecessary Complexity:** The core layer (3650s) is overly meshed (12+ redundant links), increasing configuration errors and potential **spanning-tree (STP) instability** if not managed properly.  
  - **No Security Boundaries:** The network lacks clear "zones" (e.g., DMZ for servers), violating **defense-in-depth** principles.  

---

### **5. Recommendations**  
1. **Implement a Firewall and DMZ (Critical):**  
   - Deploy a firewall (e.g., Cisco ASA) between the core layer and server segment to isolate Server1–2 and DHCP-Server into a **DMZ**. This enforces strict access control for Internet-facing services (e.g., DHCP, web) and prevents internal users from directly accessing servers.  
   - *Why?* This addresses the critical security gap and aligns with Cisco’s "3-Tier Hierarchical Model" (core, distribution, access with security at the distribution layer).  

2. **Introduce VLANs and Trunking (Immediate):**  
   - Segment the network using VLANs:  
     - VLAN 10 (PCs), VLAN 20 (IP Phones), VLAN 30 (Servers), VLAN 40 (DHCP).  
     - Configure 802.1Q trunking on core switches to separate traffic at Layer 2.  
   - *Why?* Reduces broadcast domains, isolates sensitive traffic (e.g., VoIP), and mitigates security risks like ARP poisoning.  

3. **Add Redundant DHCP and STP Optimization:**  
   - Deploy a **second DHCP server** (in HA mode) and configure it as a backup via **DHCP failover** (e.g., Cisco IOS DHCP relay).  
   - **Tune STP:** Enable **Rapid PVST+** on all 3650s to prevent loops in the meshed core and reduce convergence time.  
   - *Why?* Eliminates the DHCP SPOF and stabilizes the redundant core layer.

The recommendations focus on **security-first principles** (NIST SP 800-53, Cisco Best Practices) while preserving existing redundancy.

### **4. Weaknesses & Risks (Cons)**
❌ **Single Points of Failure (SPOF)**:
   - If **Switch0** fails, PCs (PC0, PC1, PC2) lose connectivity to the Internet and each other.
   - If the **Router1** fails, the entire network loses Internet access.
   - The **linear switch connection** (Switch0 → Switch1 → Switch2) creates an SPOF between switches.

❌ **No Redundancy**:
   - No backup links (e.g., no EtherChannel, HSRP, or VRRP).
   - No failover mechanisms for critical devices.

❌ **Lack of Security Boundaries**:
   - No firewall or segmentation (e.g., VLANs, ACLs, or a DMZ for the server).
   - All devices are on the same broadcast domain, increasing attack surface (e.g., ARP spoofing, lateral movement).

❌ **Inefficient Scalability**:
   - Adding more switches or devices requires manual configuration.
   - No hierarchical design (e.g., Core-Distribution-Access model) for future growth.

❌ **Potential Broadcast Storms**:
   - Since all switches are in the same Layer 2 domain, broadcast traffic (e.g., ARP, DHCP) floods the entire network.

---

### **5. Recommendations (Industry Best Practices)**
#### **1. Implement a Hierarchical Design (Cisco 3-Tier Model)**
   - **Core Layer**: Replace the linear switch chain with a **redundant core switch** (e.g., StackWise or VSS).
   - **Distribution Layer**: Introduce a **distribution switch** (if scaling beyond a small network).
   - **Access Layer**: Keep endpoint switches (2960-24TT) but ensure **VLAN segmentation** (e.g., separate VLANs for PCs, servers, and IoT if applicable).
   - **Redundant Links**: Use **EtherChannel (LACP)** between switches to eliminate SPOFs.

#### **2. Introduce Security Segmentation & Firewall Rules**

     - Isolate the **Server0** in a separate VLAN (e.g., VLAN 10 for servers).
     - Place PCs in a different VLAN (e.g., VLAN 20 for workstations).
   - **Router ACLs/Firewall**:
     - Implement **access control lists (ACLs)** on Router1 to restrict unnecessary traffic.
     - If possible, add a **basic firewall (e.g., ASA, pfSense)** between the Internet and internal network for deeper inspection.
   - **DMZ for Server (if public-facing)**:

#### **3. Add Redundancy & High Availability (HA)**

     - Deploy a **second router** (e.g., HSRP/VRRP) for failover.
   - **Switch Redundancy**:
